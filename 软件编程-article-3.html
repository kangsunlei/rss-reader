从单个 DLT 管道发布到多个目录和模式

3/12/2025, 4:00:00 PM

📌 一句话摘要

Databricks DLT 管道现在支持将表发布到多个目录和模式，从而简化数据工程流程并降低运营成本。

📝 详细摘要

本文宣布增强了 Databricks DLT (Delta Live Tables) 管道。现在，它们能够将表发布到单个管道中的多个目录和模式。这简化了数据管理，降低了运营复杂性，并通过将奖牌架构 (Bronze、Silver、Gold) 整合到一个管道中来降低成本。此增强功能支持完全和部分限定的表名，以及类似于标准 SQL 的 `USE SCHEMA` 和 `USE CATALOG` 命令。它还通过允许将事件日志作为 Unity Catalog 元数据存储中的标准表发布来提高可观察性，从而进一步降低基础设施开销和成本。本文提供了有关如何通过 UI、API 和 Databricks 资产包 (DABs) 开始使用此功能的说明，以及有关如何在 SQL 和 Python 中定义和引用表的示例。

💡 主要观点

  1. DLT 管道现在支持发布到多个目录和模式。 此增强功能通过将奖牌架构整合到单个管道中来简化数据管理，从而降低运营复杂性和成本。
  2. SQL 和 Python 中支持完全和部分限定的表名。 该功能支持标准 SQL 命令（如 `USE SCHEMA` 和 `USE CATALOG`），从而可以更轻松地跨多个目录和模式定义和引用表。
  3. 与 Databricks 资产包 (DABs) 集成可用。 用户可以使用 Databricks 资产包配置和部署具有多个目录和模式支持的 DLT 管道，从而简化部署过程。
