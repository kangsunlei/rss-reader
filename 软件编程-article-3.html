凉凉了吗？谈向量数据库的兴与衰

2/22/2025, 4:07:16 PM

📌 一句话摘要

文章分析了向量数据库因 RAG 应用兴起而发展，又因与传统搜索融合而面临衰落的原因和趋势。

📝 详细摘要

文章回顾了向量数据库从兴起到逐渐与传统搜索引擎融合的发展历程。最初，嵌入技术的普及催生了对向量数据库的需求，Pinecone 等公司引领了向量数据库的潮流。随着 RAG 应用的兴起，向量数据库经历了爆炸性增长。然而，随着 Elasticsearch 等传统搜索引擎整合向量搜索能力，以及向量数据库提供商开始增加传统搜索功能，市场迅速演变。文章作者认为，向量数据库的本质是具备向量功能的搜索引擎，向量搜索应作为工具箱的一部分，而非独立类别，未来的趋势是向量搜索与传统搜索的融合。

💡 主要观点

  1. 嵌入技术的普及是向量数据库兴起的根本原因 嵌入技术使得开发者能够构建以前只有大型科技公司才能实现的功能，强大的预训练模型和 API 的可访问性降低了技术门槛，从而催生了对向量数据库的需求。
  2. 向量数据库的本质是具备向量功能的搜索引擎 向量搜索是现代搜索工具箱中的一个强大工具，而不是一个独立的类别，因此向量数据库最终会与传统搜索引擎融合。
  3. 高质量的信息检索需要结合多种检索和排名策略 简单的向量相似性计算无法满足实际需求，需要结合精细的排名机制、相关性调优和经过实战测试的文本匹配算法。
  4. 单纯的向量搜索不足以满足实际的现实世界需求 现实世界的应用往往需要的不仅仅是相似性搜索，还需要过滤、分面和文本搜索等功能。Elasticsearch 通过整合向量搜索能力，表明未来的搜索在于结合传统搜索功能与嵌入技术。

💬 文章金句

  • 我们称之为 "向量数据库" 的，实际上是具有向量功能的搜索引擎。
  • 向量搜索只是工具箱中的一个强大工具，而不是一个独立的类别。
  • 没有理由用普通数据库来驱动电商搜索，添加向量功能也不会突然将 SQL 数据库变成一个完整的 "检索引擎"。
