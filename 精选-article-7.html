微软 GUI 智能体 OmniParser 二代开源！推理延迟降低 60%，大模型玩手机更溜了

3/7/2025, 5:42:42 AM

📌 一句话摘要

微软开源 OmniParser V2，通过结构化 UI 元素显著提升大语言模型在 GUI 自动化中的性能和效率，并降低推理延迟达 60%。

📝 详细摘要

文章介绍了微软开源的 OmniParser V2，旨在解决通用大语言模型（LLM）在图形用户界面（GUI）自动化中面临的挑战，如识别可交互图标和理解界面元素语义。OmniParser V2 通过将 UI 截图转换为 LLM 可解释的结构化元素，弥合了像素空间和语义理解之间的差距。该版本在检测小图标和推理速度上显著提升，延迟降低 60%。OmniParser V2 还与 GPT-4o 等多种 LLM 结合，在 ScreenSpot Pro 等基准测试中表现出色，尤其是在高分辨率屏幕和极小目标图标的 ScreenSpot Pro 基准测试中，准确率提升显著。为了方便实验，微软还提供了 OmniTool，一个集成了必要工具的 Docker 化 Windows 系统。文章还提到了微软在 AI 伦理和安全方面的考虑，以及相应的风险缓解措施。

💡 主要观点

  1. OmniParser V2 显著提升 LLM 在 GUI 自动化中的性能 通过结构化 UI 元素，降低 LLM 理解和操作 GUI 的难度，提高自动化任务的准确性和效率。
  2. OmniParser V2 通过采用更大规模的数据集和优化模型设计，显著提升了小图标检测的准确率，并将推理延迟降低了 60% 更大规模的数据训练和优化的模型设计，使得 OmniParser V2 在处理复杂 UI 界面时更加高效。
  3. OmniTool 简化了不同智能体设置的实验流程 Docker 化的 Windows 系统集成了 OmniParser 和多种 LLM，方便研究人员快速进行实验和评估。

💬 文章金句

  • OmniParser 通过将 UI 截图从像素空间「token 化」为 LLM 可解释的结构化元素，弥合了这一差距
  • 在 UI 截图的视觉提示基础上，加入局部边界框的语义信息（以文本提示的形式）能够显著提升 GPT-4o 的理解效果
  • OmniParser V2 的推理延迟比上一版本降低了 60%
