📌 一句话摘要

尤雨溪在 D2 大会上分享了 Vite 的现状、架构问题以及未来统一工具链 Void 的规划，包括 Rolldown、OXC 等关键技术。

📝 详细摘要

本文总结了尤雨溪在 D2 终端技术大会上关于 Vite 前端统一工具链的演讲内容。文章回顾了 Vite 的发展历程，肯定了其在前端生态中的地位，以及作为下一代 web 应用共享基建层的重要作用。文章分析了 Vite 当前架构存在的问题，如依赖重叠、生产构建速度慢、性能瓶颈和高阶功能缺失等。为了解决这些问题，Vite 团队提出了下一代 JS 统一工具链 Void 的构想，其核心是利用 Rust 构建的 Rolldown 和 OXC。文章详细介绍了 Rolldown 的目标和 OXC 的进展，并规划了 Vite 未来几个版本的演进路线，最终目标是推出 Vite Plus，提供一站式工具链解决方案。

💡 主要观点

  1. Vite 已成为下一代 Web 应用的共享基建层 Vite 在开发者中受欢迎程度高，生态庞大且多元化，已被众多公司应用于生产环境，为构建高性能 Web 应用提供了坚实的基础。
  2. Rolldown 旨在解决 Vite 的性能瓶颈和功能缺失 通过兼容 Rollup 插件 API，利用 Rust 提升构建性能，并添加 Rollup / esbuild 不具备的高阶功能，从而优化 Vite 的构建流程。
  3. Void 目标是构建统一的 JS 工具链 解决 JS 工具生态碎片化问题，提供包括 Vite Dev server、Rolldown 打包工具、Oxc 语言工具链和 Vitest 测试工具等全方位解决方案。

💬 文章金句

  • Vite 已经成为了 下一代 web 应用的**共享基建层。
  • Vite 面对的问题其实是整个 JS 工具生态一直存在的问题的缩影。
  • Vite Plus 作为 "cargo for JavaScript"，提供 Dev / Build、Test / Bench、Lint / Format 等一站式工具链解决方案
