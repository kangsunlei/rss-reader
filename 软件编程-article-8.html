不要再写满屏 import 导入啦！

2/8/2025, 9:28:52 AM

📌 一句话摘要

本文介绍了多种优化 import 语句的方法,旨在解决代码中 import 语句过多的问题,包括模块重导、require.context、动态导入、ProvidePlugin、TS 命名空间和 babel 特性等。

📝 详细摘要

文章针对代码中 import 语句过多的问题,从多个角度给出了优化方案。首先介绍了模块重导的概念和使用方式,通过 Re-export 简化导入路径,提高代码可读性和可维护性。其次,讨论了 require.context 在动态导入模块方面的应用,尤其是在处理路由配置时的优势。接着,提到了动态 import 和 webpack.ProvidePlugin 的使用,以及它们各自的优缺点,例如 ProvidePlugin 虽然方便,但可能隐藏依赖关系。此外,文章还介绍了如何利用 Typescript 命名空间减少 import 语句,以及如何通过配置 babel 来优化 React 项目中的 import。最后,文章还分享了设置 webpack 别名(不仅缩短路径,还能提高代码可维护性)、格式化 prettier.printWidth、动态全局加载组件和使用 babel-plugin-import 等其他优化技巧。旨在帮助开发者编写更简洁、更易于维护的代码。

💡 主要观点

  1. 模块重导可以简化导入路径,提高代码可读性 通过在 index.ts 文件中统一暴露模块,可以避免在其他文件中写大量的 import 语句,从而简化代码。
  2. require.context 适用于动态导入一组模块的场景 尤其是在配置路由时,可以避免手动编写大量的 import 语句,提高开发效率,减少出错的可能性。
  3. webpack.ProvidePlugin 可以减少重复的导入语句 通过配置全局变量,可以在任何地方使用指定的变量 / 函数 / 库,而无需手动导入,从而提高开发效率,但也可能隐藏依赖关系。
  4. Typescript 命名空间可以在 TS 项目中减少 import 的导入 通过将类型定义在命名空间中,可以在使用时无需显式导入类型,从而减少 import 语句的数量。
  5. babel-plugin-import 的使用可以通过优化 import 语句来减少打包体积 对于使用了大型第三方库的项目来说,这是一种非常有价值的优化手段。

💬 文章金句

  • 如果打开一个文件,看到的全是满屏幕的 **「import」** 语句,这将是一种怎样的体验?
  • Re-export 一般用于收拢同类型的模块、一般都是以文件夹为单位,如 components、routes、utils、hooks、stories 等都通过各自的 index.tsx 暴露,这样就能极大程度的简化导入路径、提升代码可读性、可维护性。
  • 在 TS 项目中,满屏 import 肯定少不了 TS 的份。
  • 同样,它和`ProvidePlugin`一样炸裂,可以直接灭掉`import`导入。
