📌 一句话摘要

微软使用 Go 重构 TypeScript 编译器，由 C# 之父操刀，性能提升 10 倍。

📝 详细摘要

为了解决 TypeScript 编译器在大规模代码库中的性能瓶颈，微软决定采用 Go 语言对 TypeScript 编译器和工具链进行原生重构，性能提升高达 10 倍，内存占用降低约 50%。 此次重构由 C# 和 TypeScript 之父 Anders Hejlsberg 亲自操刀。 选择 Go 语言的原因在于其编码风格与现有 TypeScript 代码库相似，便于代码迁移，同时能够精细控制内存布局。 微软计划在 2025 年中期预览 TypeScript 编译器的本机实现，并在年底完整支持项目构建和编辑器语言服务。 这一举措引发了关于编程语言选型的讨论，开发者对微软为何不选择 C# 或 Rust 提出了质疑。

💡 主要观点

  1. 原生 Go 重构显著提升 TypeScript 编译器性能 通过 Go 语言重构，编译速度提升 10 倍，编辑器响应更快，内存占用更低，从而改善大型代码库的开发体验。
  2. 微软选择 Go 语言是出于代码迁移和兼容性的考虑 Go 的编码风格与现有 TypeScript 代码库相似，便于代码迁移，且能精细控制内存布局，满足 TypeScript 的需求。
  3. 微软的选型引发了关于编程语言适用性的激烈讨论 开发者质疑为何不选择 C# 或 Rust，认为微软错失了整合 Web 开发生态的机会。
