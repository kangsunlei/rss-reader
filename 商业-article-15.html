我们对于 GPU 的看法错了 [译]

2/17/2025, 5:11:42 AM

📌 一句话摘要

Fly.io 反思其 GPU 云服务策略的失败，并调整方向，转向更符合市场需求的 LLM API 服务。

📝 详细摘要

Fly.io 最初押注 AI/ML 推理对 GPU 的需求，推出了 Fly GPU Machines 云服务。然而，市场反馈表明，开发者更倾向于使用 OpenAI 或 Anthropic 等提供的 LLM API，而非自行搭建 GPU 环境。Fly.io 为此投入了大量资源，包括专用服务器硬件、高昂的安全评估费用和虚拟化技术研究，但并未获得预期的市场反响。文章总结了 Fly.io 在 GPU 策略上的经验教训，包括对 AI/ML 发展趋势的误判，以及坚持面向广泛开发者需求的核心原则。尽管 GPU 业务未成爆款，但 Fly.io 从中获得的资产和经验，以及对安全隔离和开发者体验的坚持，仍然具有重要价值。未来，Fly.io 可能会转向更专注于 LLM API 相关的云服务。

💡 主要观点

  1. 大部分开发者需要的是调用 LLM API 的能力，而不是自己维护 GPU 环境 大多数开发者只想让应用调用大型语言模型，并不想自己搭建 GPU 环境，这使得 Fly.io 的 GPU 云服务未能获得广泛市场。
  2. 对 AI/ML 发展趋势的误判是 GPU 策略失败的原因之一 Fly.io 最初认为会出现各种主流模型，但现实是 LLM 占据了主导地位，这使得对 GPU 的需求主要集中在少数几家公司。
  3. 坚持面向广泛开发者需求的核心原则至关重要 Fly.io 坚持做核心特性时要面向 1 万个开发者，而不是只迎合少数特定客户，这使得其在 GPU 策略上保持了对主产品的关注。
  4. 高额安全投入并没有带来预期的市场回报 为确保 GPU 云服务的安全性，Fly.io 投入了大量资源进行安全评估和技术研究，但这些投入并没有转化为市场优势。

💬 文章金句

  • 我们选择了人迹罕至的道路，但后来才发现它之所以罕至，是有原因的。
  • 开发者并不真正想要 GPU 。他们甚至不真正想要 AI/ML 模型，他们要的是 LLM 。
  • 我们对 GPU 的看法是错了，但这正是我们发现正确道路的方式。
