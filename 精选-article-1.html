OpenAI 突然发布智能体 API！支持网络和文件搜索以及 computer use

3/12/2025, 2:35:44 AM

📌 一句话摘要

OpenAI 发布核心 Responses API，内置网络 / 文件搜索和计算机使用能力，配合 Agents SDK 等工具，大幅简化智能体构建流程。

📝 详细摘要

OpenAI 发布了一系列新工具，旨在简化智能体的构建过程。核心是 Responses API，它整合了 Chat Completions API 的简洁性和 Assistants API 的工具使用能力，内置了网络搜索、文件搜索和计算机使用（computer use）等功能。此外，还发布了 Agents SDK，提供智能体配置、交接、安全检查和可观察性等功能，用于编排单智能体和多智能体工作流，以及用于跟踪和检查智能体工作流程执行情况的可观测性工具。Responses API 简化了 OpenAI 模型和工具的集成，提供了更灵活的智能体构建基础，使开发者能够更轻松地将 OpenAI 模型和内置工具集成到应用中。OpenAI 计划在未来逐步淘汰 Assistants API，并提供迁移指南。同时，OpenAI 还开源发布了 Agents SDK，简化多智能体工作流的编排。Responses API 的设计理念是交付能力，而非抽象。

💡 主要观点

  1. Responses API 是 OpenAI 构建智能体的核心方向 Responses API 既有 Chat Completions 的简洁性，又具备 Assistants API 的工具使用能力，降低了智能体开发的门槛，是 OpenAI 构建智能体的未来方向。
  2. 内置工具如网络搜索和文件搜索增强了智能体的能力 通过内置工具，智能体可以直接访问网络信息和文件内容，从而更好地完成各种复杂任务。
  3. Agents SDK 简化了多智能体工作流的编排 Agents SDK 提供了一套易于配置的工具和接口，帮助开发者更轻松地构建和管理多智能体系统。

💬 文章金句

  • Responses API 代表了在 OpenAI 上构建智能体的未来方向。
  • 交付能力，而非抽象。（Ship capabilities， not abstractions.）
  • 只需 4 行代码即可开始使用，只需一个参数即可包含文件搜索、网络搜索、函数调用和结构化输出等功能。
