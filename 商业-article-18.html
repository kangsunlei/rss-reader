本文介绍了 Cursor IDE 中 Agent 模式下 Claude 3.5 Sonnet AI 编码助手的系统提示词规范。

📝 详细摘要

本文深入剖析了 Cursor IDE 中 Agent 模式下，由 Claude 3.5 Sonnet 驱动的 AI 编码助手的系统提示词。该提示词定义了 AI 助手的核心角色，即作为一位强大的、具有自主能力的编码助手，与用户进行结对编程，旨在引导 AI 助手高效、安全地协助用户完成各种编码任务，并确保生成的代码质量。文章详细阐述了 AI 助手在工具调用、信息检索、代码变更以及外部 API 调用等方面的行为准则，同时也提及了对用户查询 (user_query) 的处理。此外，文章还提及了用户操作系统、工作空间路径等信息，以便 AI 助手更好地理解用户环境。总体而言，该系统提示词旨在规范 AI 助手在 Cursor IDE 中的行为，使其更有效地辅助用户完成编码任务。该提示词的设计在很大程度上决定了 AI 助手行为是否符合预期，以及最终代码的质量。

💡 主要观点

  1. AI 助手需严格遵守工具调用规范，确保参数完整和避免无效调用。 AI 助手在调用工具时，必须遵循预定义的工具调用模式，提供所有必要参数，并且避免调用已失效的工具，以确保工具的有效使用。
  2. 代码变更需遵循严格规范，确保生成代码可立即运行。 AI 助手在进行代码变更时，需要添加所有必要的 import 语句、依赖项和端点，创建合适的依赖管理文件，并修复 linter 错误，以确保生成的代码可以直接运行。这些规范旨在提高代码质量，减少用户调试时间。
  3. 充分利用外部 API，无需事先征得用户同意，但需注意 API Key 的安全。 AI 助手可以根据任务需要，选择最合适的外部 API 和软件包来解决问题，无需事先征得用户同意，但需要注意 API Key 的安全问题，避免硬编码。

💬 文章金句

  • You are a powerful agentic AI coding assistant， powered by Claude 3.5 Sonnet. You operate exclusively in Cursor， the world's best IDE.
  • ALWAYS follow the tool call schema exactly as specified and make sure to provide all necessary parameters.
  • When making code changes， NEVER output code to the USER， unless requested.
  • Unless explicitly requested by the USER， use the best suited external APIs and packages to solve the task. There is no need to ask the USER for permission.
