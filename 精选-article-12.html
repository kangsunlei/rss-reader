Gemini API：先进的文本嵌入技术

3/7/2025, 12:00:00 AM

📌 一句话摘要

Google 通过 Gemini API 发布实验性 Gemini 文本嵌入模型，凭借增强的语义理解能力在 MTEB 多语言排行榜上表现出色，是 RAG 应用的理想之选。

📝 详细摘要

Google 通过 Gemini API 推出了新的实验性 Gemini 文本嵌入模型 (gemini-embedding-exp-03-07)，旨在显著提升文本嵌入的质量与效率。该模型基于 Gemini 模型进行训练，使其在语言理解和细致语境分析方面表现出色，适用于各种应用场景。该模型在 MTEB 多语言排行榜上领先，优于之前的 text-embedding-004 模型，并支持更长的输入 token 长度。它在金融、科学和法律领域表现良好，无需进行大量的微调。该模型具备 8K 的输入 token 限制、3K 的输出维度、对 100 多种语言的支持以及统一的架构，超越了以往特定于任务的模型。目前该模型尚处于实验阶段，容量有限。Google 鼓励开发者积极探索其各项功能，并提供宝贵反馈。

💡 主要观点

  1. Gemini 文本嵌入模型在 MTEB 多语言排行榜上领先 该模型在检索、分类等多种任务中表现优异，在 MTEB 多语言排行榜上获得平均任务得分 68.32，领先第二名 5.81 分。
  2. Gemini 文本嵌入模型具有更长的输入 token 长度和更高的输出维度 该模型具备 8K 输入 token 限制和 3K 输出维度，约为以往嵌入模型的 4 倍，能够处理更大规模的文本、代码等数据。
  3. Gemini 文本嵌入模型支持 Matryoshka Representation Learning (MRL) MRL 技术允许用户截断原始的 3K 维度，进而降低存储成本，并根据实际需求提供灵活调整空间。
  4. Gemini 文本嵌入模型是一种统一的模型，超越了以往特定于任务的模型 该模型的质量优于 Google 以前的特定于任务的多语言、仅英语和特定于代码的模型，从而提供了更通用的解决方案。

💬 文章金句

  • 该嵌入模型基于 Gemini 模型训练而成，承袭了 Gemini 对语言和细致语境的深刻理解，因此可广泛应用于各种场景。
  • Gemini 文本嵌入模型取得了平均任务得分 68.32，领先于下一个竞争模型 5.81 分。
  • MRL 技术允许用户截断原始的 3K 维度，进而降低存储成本。
