防御利用 SSE-C 加密的 S3 勒索软件攻击

2/23/2025, 12:00:00 AM

📌 一句话摘要

AWS 发布措施应对利用 SSE-C 加密亚马逊 S3 存储桶的勒索软件。

📝 详细摘要

Codefinger 勒索软件活动利用泄露的 AWS 凭证，通过服务器端加密（SSE-C）特性加密亚马逊 S3 数据，并勒索解密密钥。AWS 建议使用短期凭证，实施数据恢复程序，并在不需要时阻止在 S3 存储桶上使用 SSE-C。Halcyon 和 AWS 建议限制 SSE-C 使用、审计 AWS 密钥、启用高级日志记录并及时联系 AWS 支持。该事件强调了云安全中凭证管理和最小权限原则的重要性。

💡 主要观点

  1. Codefinger 利用 SSE-C 特性加密 S3 数据进行勒索。 攻击者获得具有 GetObject （获取对象）和 PutObject （上传对象）权限的 AWS 密钥，使用 SSE-C 加密 S3 对象，并要求支付赎金以获取解密密钥。
  2. AWS 建议使用短期凭证和数据恢复程序。 为了减轻风险，AWS 强调使用短期凭证，实施数据恢复程序，并在不需要时阻止在 S3 存储桶上使用 SSE-C。
  3. 限制 SSE-C 使用、审计密钥、启用高级日志是缓解措施。 Halcyon 和 AWS 建议立即限制 SSE-C 使用，审计 AWS 密钥，启用高级日志记录，并迅速联系 AWS 支持。

💬 文章金句

  • 与传统勒索软件在本地或传输过程中加密文件不同，此攻击直接与 AWS 的安全加密基础设施集成。加密后，如果没有攻击者的密钥，恢复是不可能的。
  • “加密一切” —— 但要适度！这反映了 AWS 安全策略的复杂性，用户需谨慎配置。
  • 如果您的应用程序不使用 SSE-C 作为加密方法，您可以使用应用于 S3 存储桶的资源策略或应用于亚马逊云科技组织的资源控制策略 (Resource Control Policy， RCP) 来阻止使用 SSE-C。
