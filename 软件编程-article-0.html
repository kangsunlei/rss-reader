这些很新的 JS 数组方法，还挺好用！

2/9/2025, 11:33:41 AM

📌 一句话摘要

介绍了 7 个 JavaScript 数组新增方法，主要提供非破坏性的元素访问、替换、排序和查找功能，保证原数组不变。

📝 详细摘要

本文详细介绍了近三年添加到 JavaScript 数组原型中的 7 个新方法，分别是 `at()`、`with()`、`toSorted()`、`toReversed()`、`toSpliced()`、`findLast()` 和 `findLastIndex()`。`at()` 方法支持负索引访问数组元素。`with()` 方法可以在不修改原数组的情况下替换指定索引的元素。`toSorted()`、`toReversed()` 和 `toSpliced()` 方法分别提供了不修改原数组的排序、反转和删除/替换元素的功能，对应于 `sort()`、`reverse()` 和 `splice()`，适用于函数式编程。`findLast()` 和 `findLastIndex()` 方法则允许查找数组中最后一个满足条件的元素及其索引。文章通过代码示例和浏览器兼容性截图，清晰地展示了这些新方法的用法和优势，主要解决在数据处理或状态管理中，不修改原数组的前提下操作数组的需求。

💡 主要观点

  1. `with()` 以及 `toSorted` 等方法保证了数组操作的不可变性，但需注意浏览器兼容性 这些方法返回数组的新副本，确保在修改数组时不会影响原始数据，这在函数式编程和状态管理中非常有用。但在实际应用中，需要注意目标用户的浏览器是否支持这些新方法。
  2. `at()` 方法简化了数组末尾元素的访问 通过负索引，`at()` 方法提供了一种更简洁的方式来访问数组的最后一个或倒数第 n 个元素，避免了手动计算索引的麻烦。
  3. `findLast()` 和 `findLastIndex()` 增强了数组查找的灵活性 这两个方法允许从数组的末尾开始查找元素，适用于需要查找最后一个匹配项的场景，提高了查找效率。

💬 文章金句

  • 通常情况下，我们会使用 `[index]` 索引来获取数组元素，但这种方式获取正数元素还可以，如果想要获取倒数元素，就麻烦一点，这时就可以借助 `at()` 方法来实现，只需指定一个负数即可。
  • 当需要更新数组中特定位置的元素但不想修改原数组时，可以使用 `with()` 方法。
  • 当需要对数组进行排序但不想修改原数组时，可以使用 `toSorted()` 方法。
  • 当需要反转数组但不想修改原数组时，可以使用 `toReversed()` 方法。
  • 当需要从数组中删除或替换某些元素但不想修改原数组时，可以使用 `toSpliced()` 方法。
