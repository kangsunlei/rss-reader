Databricks 发布自动 Liquid Clustering

3/5/2025, 8:00:00 PM

📌 一句话摘要

Databricks 发布自动 Liquid Clustering 公共预览版，通过预测式优化自动优化数据布局，提升 SQL 查询性能并降低成本。

📝 详细摘要

Databricks 发布自动 Liquid Clustering，旨在解决手动数据布局的复杂性和低效性。该功能利用预测式优化和 Unity Catalog 自动管理数据布局，无需手动调整。用户只需设置 `CLUSTER BY AUTO`（使用 AUTO 进行聚类），系统即可根据查询模式智能选择聚类键并自动聚类，优化 SQL 查询性能。该功能通过收集遥测数据、建模工作负载和进行成本效益分析，动态调整聚类方案，显著提升查询速度并降低总拥有成本。Healthrise 和 CFC Underwriting 等预览客户报告了显著的性能提升和成本降低。

💡 主要观点

  1. 选择最佳数据布局是一个难题 传统数据布局方案（如分区和 ZORDER）在避免数据倾斜和并发冲突方面存在困难，管理复杂且成本高昂。自动 Liquid Clustering 旨在解决这些痛点。
  2. 自动 Liquid Clustering 简化数据管理，无需手动调整数据布局 通过预测式优化和 Unity Catalog，系统自动监控和分析数据和查询模式，智能选择聚类键，优化数据组织，提升 SQL 查询性能。
  3. 预测式优化通过成本效益分析动态调整聚类方案 系统收集查询扫描统计数据，建模工作负载，评估不同聚类方案的性能收益，并在性能提升超过成本时更新聚类键。
  4. 自动 Liquid Clustering 显著提升 SQL 查询性能并降低总拥有成本 预览客户报告查询速度提升高达 10 倍，并通过减少手动维护工作降低了计算成本和总拥有成本。
