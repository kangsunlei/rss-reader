DLT Sink API 介绍：将数据写入 Kafka 和外部 Delta Lake 表

2/17/2025, 5:00:00 PM

📌 一句话摘要

Databricks 推出 DLT Sink API，以简化将数据写入 Kafka 和 Delta Lake 表的过程，从而实现与外部系统的无缝集成。

📝 详细摘要

本文介绍了 Databricks 中的 DLT Sink API，该 API 简化了将处理后的数据写入外部系统（如 Apache Kafka、Azure Event Hubs 和 Delta Lake 表）的过程，以实现更灵活的数据集成和实时分析。该 API 通过允许用户轻松配置 Event Hubs 和 Delta Sink，从而简化了数据管道的配置和管理。它通过促进将数据无缝流式传输到外部系统，从而增强了 DLT 管道在实时分析和事件驱动架构中的价值。本文提供了有关如何使用 `create_sink` API 配置 Sink 以及如何使用 `append_flow` 装饰器来增强功能的示例。它还重点介绍了对 `applyInPandasWithState` 的支持，以便在 DLT 管道中进行有状态处理，从而进一步增强数据转换和聚合功能。

💡 主要观点

  1. DLT Sink API 使得向外部系统写入数据更为便捷，从而提高了数据集成灵活性。 新的 API 允许 DLT 管道与 Kafka 和 Delta Lake 表等外部事件流无缝集成，从而扩大了 DLT 的范围并简化了配置。
  2. `create_sink` API 通过其简洁的设计简化了 Sink 配置。 `create_sink` API 允许用户轻松配置 Event Hubs 和 Delta Sink，并提供格式规范和特定于 Sink 的设置选项，从而使其易于使用。
  3. `append_flow` 装饰器可以便捷地将连续数据流式传输到 Sink 中。 增强的 `append_flow` API 能够将流式数据传输到配置的 Sink 中，从而确保使用最新信息进行持续更新。
  4. `applyInPandasWithState` 的支持显著增强了有状态处理能力。 `applyInPandasWithState` 函数现在在 DLT 中受支持，使用户能够利用 Pandas 的强大功能在其 DLT 管道中进行有状态处理，这是 DLT 功能的重大增强。

💬 文章金句

  • DLT 中引入的 Sinks API 旨在解决数据写入问题。它使用户能够将处理后的数据写入外部事件流，例如 Apache Kafka、Azure Event Hubs 和 Delta Lake 表。
  • Sink API 在 `dlt` Python 包中可用(可调用)，可以与 `create_sink()` 一起使用
  • [@append_flow](https://docs.databricks.com/en/delta-live-tables/flows.html#use-append-flow-to-write-to-a-streaming-table-from-multiple-source-streams) API 得到了增强，允许将数据写入由 Sink 名称指定的目标 Sink 中。
