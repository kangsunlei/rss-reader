Linus 将不顾反对合并 Rust 代码？！内核维护者无奈强调：不能让“癌症”扩散！

2/21/2025, 8:08:43 AM

📌 一句话摘要

Linus 计划在 Linux 内核中合并 Rust 代码，引发维护者对多语言维护负担的担忧。

📝 详细摘要

本文围绕 Linux 内核引入 Rust 代码的争议展开，尽管 Rust 在内存安全方面有优势，但内核维护者 Christoph Hellwig 担忧多语言混编增加维护难度，甚至将其比作“癌症”，主张在现有 C 代码上加强防护。另一维护者 Greg K-H 则支持引入 Rust，认为可减少 C 语言中常见的内存安全漏洞。Linus Torvalds 在此事中态度强硬，引发社区对治理模式的讨论，Hector Martin 因此离职，也反映了开源社区面临的维护者倦怠等挑战。

💡 主要观点

  1. Linus 或不顾反对合并 Rust 代码，引发内核维护者担忧 Linus Torvalds 的态度对 Linux 内核发展方向具有重要影响，但可能与部分维护者的意见相左，引发社区内部的讨论和争议。
  2. 多语言混编增加维护难度，可能导致代码库混乱 在 C 和 Rust 之间互操作需要大量的绑定层，分散到各个子系统可能造成核心基础设施的重复改写与无法预期的复杂度。
  3. Linus 的强硬态度和社区治理模式引发维护者倦怠和离职 Linus 的“放任”态度让子系统维护者可以阻挠并滥用其他贡献者，打击社区士气却毫无后果，导致维护者倦怠甚至离职。

💬 文章金句

  • 所以从现在开始，作为一名 Linux 开发者或维护者，你想接受 Rust 也好，不想接受也罢，都必须去面对它。
  • 再添加一种新语言不应该成为问题。
  • 我们遇到的大多数（按数量而不是质量或严重程度来算）漏洞，都是源于 C 中那些愚蠢的小角落，而这些问题在 Rust 中完全不存在
  • 但是对于新的代码和驱动来说，用 Rust 编写可以杜绝（或大大减少）这些类型的错误，这对我们所有人来说都是好事，我们为什么不这么做呢？
