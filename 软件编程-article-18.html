Docker 引擎 v28：默认增强容器网络安全性

2/28/2025, 9:04:49 AM

📌 一句话摘要

Docker 引擎 v28 通过默认增强容器网络安全性，防止本地网络中的意外访问。

📝 详细摘要

Docker 引擎 v28 引入了默认增强的容器网络安全策略，旨在减少容器在本地网络上的意外暴露风险。此版本通过默认丢弃到未发布端口的流量，增强了安全性。文章解释了先前版本中可能存在的潜在风险，即在特定网络配置下，未发布的容器端口可能被同一局域网内的其他主机访问。新版本通过调整 iptables 规则，默认阻止此类访问，同时提供了灵活的退出机制，允许依赖直接容器 IP 访问的场景根据需要恢复旧的行为。文章还提供了详细的升级指南和故障排除建议，以帮助用户顺利过渡到新版本。

💡 主要观点

  1. Docker 引擎 v28 默认丢弃到未发布端口的流量，增强了安全性 新版本通过调整 iptables 规则，默认阻止同一局域网内的其他主机访问未发布的容器端口，降低了潜在的安全风险。
  2. 用户可以通过配置选项选择禁用新的安全策略 如果用户依赖于旧的行为，可以通过设置 `ip-forward-no-drop` 或创建 `nat-unprotected` 网络来恢复之前的网络配置。
  3. 升级前应仔细检查防火墙设置并进行测试 建议用户在升级前检查当前的 iptables 设置，并在受控环境中测试新版本，以确保现有应用和服务不受影响。

💬 文章金句

  • 通过强化 Docker 引擎 28.0 的默认网络规则，我们旨在降低本地网络中容器的意外暴露风险。
  • 大多数用户可以继续使用而不会中断，并且可以享受额外的安心。
  • 未发布的端口保持私有，除非您发布或明确选择停用。
