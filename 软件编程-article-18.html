DeepSeek 提示词编写技巧典藏版！

2/11/2025, 12:04:58 AM

📌 一句话摘要

本文深入介绍了 DeepSeek 提示词编写技巧,包括万能框架、细化技巧、RAG 应用和优化模型确定性的方法,旨在帮助读者高效编写和调试 Prompt,提升大模型应用效果。

📝 详细摘要

本文详细阐述了 Prompt 工程的标准化工作流,面向 Prompt 工程师和 AI 开发者。首先提出了一个 Prompt 万能框架,将 Prompt 拆解为“立角色 + 述问题 + 定目标 + 补要求”四个部分,并解释了其有效性。然后,文章深入探讨了如何细化框架,包括如何借助招聘 JD 补全角色信息,以及如何分解复杂任务。接着,文章介绍了如何通过 RAG 技术增加更多信息,包括 Few-shot 和 Memory 的应用,以及如何应对专业领域的问题。此外,文章还分享了用参数控制模型确定性以及**利用大模型进行 Prompt 优化的创新思路**等附加技巧。最后,文章强调了 Prompt 在大模型工程中的基石作用,并鼓励读者掌握 Prompt 编写技能,成为 Prompt 工程师。文章以腾讯云开发者社区为背景。

💡 主要观点

  1. Prompt 万能框架可高效完成 Prompt 初版编写 通过“立角色 + 述问题 + 定目标 + 补要求”的结构化拆分,降低 Prompt 编写难度,提升效率,并方便 Prompt 管理。
  2. RAG 技术通过检索增加信息,优化 Prompt 效果 利用 RAG 框架,可动态增加 Few-shot 示例、历史记忆和专业领域知识,弥补大模型知识短板,提升模型准确性。
  3. 参数调节和模型自优化,提升 Prompt 质量 通过调整 Temperature 和 Top-P 参数,控制模型输出的确定性;**利用 APE、APO 等算法,让大模型自动优化 Prompt 语句,实现 Prompt 工程自动化。**
  4. 利用大模型优化 Prompt 介绍 APE、APO 等算法,探索 Prompt 工程自动化,提升 Prompt 优化效率。

💬 文章金句

  • DeepSeek 的一大进步,就在于其深度思考功能,已经极大程度上地代替了提示词工程的作用。
  • 想要依赖模型参数注入知识不是可行的做法,要让模型拥有动态获取知识的能力。
  • RAG 并不是唯一的技术方案,我们不必困在此处,在条件不足的情况下,我们可以结合场景找到最合适的检索模式,践行 RAG 的思想,在输入中增加更多信息才是最核心的思想。
  • 我们要将“专业知识”翻译成“通用知识” 告诉模型大模型,让大模型更好的应对专业领域。
  • 我们认为在一个大模型工程中,“Prompt”应该起到基石般的作用,有效稳定可扩展。
  • 清晰的结构对大模型的效果有很大的影响。
