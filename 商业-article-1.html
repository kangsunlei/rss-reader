为什么 LLM 在 OCR 任务上表现糟糕

2/8/2025, 6:50:10 PM

📌 一句话摘要

分析了 LLM 在 OCR 任务中表现不佳的原因及其在实际应用中的风险。

📝 详细摘要

本文深入探讨了大型语言模型 (LLM) 在 OCR 任务中表现不佳的原因。文章指出, LLM 在处理图像时依赖高维向量嵌入, 容易丢失视觉精细度, 且基于概率的生成机制会导致幻觉, 产生与原文不符的替换,例如数值错误和信息篡改。此外, 文章还列举了 LLM 在财务、医疗等实际应用中的失败案例, 以及 Prompt 注入带来的安全风险。最后, 文章提出结合传统计算机视觉算法和视觉 Transformer 的定制化解决方案, 为提升 OCR 任务的准确性和可靠性提供了方向。

💡 主要观点

  1. LLM 处理图像时易丢失视觉精细度 LLM 依赖高维向量嵌入, 在图像转化为抽象表示时, 更注重语义信息而非每个字符的精准识别, 易忽略小数位、分隔符等细节。
  2. LLM 基于概率的生成机制会导致幻觉 LLM 倾向于输出常见词, 修正其认为的错误, 随意合并或重排信息, 在 OCR 任务中易产生看似合理但错误的替换。
  3. LLM 在财务、医疗等领域存在数据失真、自动解题和 Prompt 注入等风险。 LLM 可能导致财务及医疗数据失真, 自动解题偏离原始表述, 甚至受到 Prompt 注入攻击, 产生被污染的输出。

💬 文章金句

  • LLM 在处理复杂 OCR 时表现确实糟糕, 而且短期内似乎也不会出现显著改进。
  • 与传统 OCR 系统在不确定时往往会显著失败不同, LLM 会给出貌似 "正确" 的猜测, 却可能完全背离事实。
  • 我们团队原以为 "GPT 可以搞定一切" 的天真想法, 让我们顺藤摸瓜地深入研究了计算机视觉、ViT 架构, 以及当下大模型在 OCR 任务中的根本性限制。
