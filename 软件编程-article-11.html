Split-tgz 现已成为 CLI 归档部署的默认行为 - Vercel

2/11/2025, 1:00:00 PM

📌 一句话摘要

Vercel 默认启用 split-tgz,提升 CLI 归档部署性能并规避文件大小限制。

📝 详细摘要

Vercel 宣布将 `split-tgz` 归档部署设为 CLI 的默认行为。此举旨在提升大型项目的部署效率,上传速度提升高达 30%,并规避文件上传大小限制。此前,`split-tgz` 作为一个可选参数 `vercel deploy --archive=split-tgz` 发布。由于其稳定性和优势,Vercel 将其设为默认,意味着之前的 `split-tgz` 参数已被弃用。用户可以通过 `tgz` 选项体验 `split-tgz` 所带来的功能和优势。文章还提供了 Vercel CLI 归档部署的文档链接以及注册免费账户、联系销售和探索企业版功能的入口。`split-tgz` 通过优化文件分片和并行上传,实现了更快的上传速度。

💡 主要观点

  1. `split-tgz` 成为 Vercel CLI 归档部署的默认选项。 Vercel 确认了 `split-tgz` 的稳定性,并将其设为默认行为,简化了部署流程。
  2. `split-tgz` 提升部署速度高达 30%。 与之前的部署方式相比,`split-tgz` 能够将上传速度提升高达 30%。
  3. `split-tgz` 避免了文件上传大小限制。 使用 `split-tgz` 可以部署包含大量文件的项目,无需担心文件大小限制。

💬 文章金句

  • Archive deployments are useful for deploying large projects with thousands of files from the CLI.
  • We've confirmed `split-tgz`'s stability and made it the default behavior for `tgz`.
  • This means the separate `split-tgz` option is now deprecated as the `split-tgz` functionality and benefits power the default `tgz` option.
