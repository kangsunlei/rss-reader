ReSharper 和 Rider 2024.3.5 版本发布，修复多项 Bug | .NET 工具博客

2/7/2025, 10:48:36 AM

📌 一句话摘要

ReSharper 和 Rider 2024.3.5 版本修复 Bug，提升测试体验，并增强 AI 代码补全及 Claude 模型支持。

📝 详细摘要

本文宣布发布 ReSharper 和 Rider 2024.3.5 版本，重点是 Bug 修复和功能增强，特别是与 AI 相关的更新。Rider 的更新引入了用于 C# 云补全的新的内部模型，从而提高了可靠性和代码建议质量。Rider 现在通过 Amazon Bedrock 在 AI Assistant 中支持 Anthropic 的 Claude 模型，包括 Claude 3.5 Sonnet 和 Claude 3.5 Haiku。ReSharper 的更新解决了 .NET Framework 下 xUnit v3 测试的问题，并改进了大型测试套件的处理。

💡 主要观点

  1. Rider 通过新的 AI 模型增强 C# 代码补全，并支持 Anthropic 的 Claude 模型。 用于 C# 云补全的新内部模型提高了代码建议的可靠性和质量。此更新增加了通过 Amazon Bedrock 对最新 Claude 模型（包括 Claude 3.5 Sonnet 和 Claude 3.5 Haiku）的支持。
  2. ReSharper 修复了 .NET Framework 下的 xUnit 测试执行，并改进了测试套件处理。 此更新解决了在 .NET Framework 下运行 xUnit v3 测试的问题，并提升了执行大型测试套件时的性能。
  3. Rider 恢复了“转到类”功能。 修复了“转到类”功能，提升 Rider 的导航体验。

💬 文章金句

  • 现在，您可以在 Test Explorer 中，在 .NET Framework 下运行 xUnit v3 测试，不会出现任何问题。
  • Test Runner 现在可以无缝处理大型测试套件，确保执行过程流畅，避免中断和性能问题。
  • 用于 C# 云补全的新内部模型。
  • 此版本增加了对最新 Claude 模型（包括 Claude 3.5 Sonnet 和 Claude 3.5 Haiku）的支持。
