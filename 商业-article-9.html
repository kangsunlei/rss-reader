C、Rust 混合用被批为 Linux 的“癌症”，Rust 开发者逼 Linus“站队”失败后愤然辞职

2/8/2025, 9:14:27 AM

📌 一句话摘要

Rust 与 C 在 Linux 内核的融合引发争议，Linus 未明确表态致开发者辞职。

📝 详细摘要

文章讲述了 Linux 内核引入 Rust 语言过程中，Rust 开发者与 Linux 内核维护者之间因 DMA API 的使用展开的冲突。Rust 开发者希望通过 Rust 提高系统安全性，但内核维护者担心多种语言混合增加代码复杂性及潜在安全风险。Linus Torvalds 的回应被认为模棱两可，未能平息争议，反而加剧了社区的分裂，最终 Asahi Linux 首席开发者 Hector Martin 因不满社区管理方式而辞职。文章引发了社区对 Linus 领导方式的质疑，以及这种质疑可能对 Linux 未来发展的影响。

💡 主要观点

  1. Rust 引入 Linux 内核旨在提高安全性，但引发维护者担忧。 Rust 开发者认为 Rust 能避免 C/C++ 的内存安全问题，但维护者担忧 Rust 引入会增加代码复杂性、降低可维护性，以及潜在的安全风险，双方存在根本分歧。
  2. Linus Torvalds 未明确站队，导致事态升级。 Linus 的回应模棱两可，未能有效解决 Rust 与 C 的争议，原因可能在于他希望保持中立，避免社区进一步分裂，但也因此失去了解决问题的机会，最终引发开发者不满，导致核心开发者辞职。
  3. Rust 与 C 的融合反映了开源社区在技术选择和治理模式上的挑战。 C 语言程序员不愿改变习惯，Rust 拥护者坚持 Rust 的优势，双方的摩擦短期内难以消除，影响技术发展。同时，也暴露出开源社区在技术选型和社区治理上存在的挑战。
