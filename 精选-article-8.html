DeepSearch/DeepResearch 中最优文本段选择和 URL 重排

3/13/2025, 7:39:35 AM

📌 一句话摘要

Jina AI 通过在 DeepSearch/DeepResearch 中应用迟分算法优化文本片段选择和 URL 重排，有效提升了 LLM Agent 的回答质量。

📝 详细摘要

本文深入探讨了 Jina AI 如何通过优化文本段选择和 URL 重排来提升 DeepSearch/DeepResearch 系统中 LLM Agent 的回答质量。文章首先介绍了利用 `jina-embeddings-v3` 模型和一维卷积实现的迟分算法，用于从长网页中提取最优文本段。该方法通过滑动窗口在文本块相似度分数上移动，找到平均值最高的窗口，从而选择与问题最相关的文本片段。其次，文章讨论了如何综合考虑频率信号、路径结构、语义相关性、最后更新时间、受限内容和域名多样性等多种因素对 URL 进行重排，引导 LLM 选择最相关的 URL。最后，文章总结了 DeepSearch 自开源以来的经验，并提出未来搜索技术可能朝着两极分化的方向发展，即轻量级 SQL/BM25 负责检索输入，LLM 负责检索输出，而中间层模型则专注于上下文窗口内的过滤、去重、排序等任务。

💡 主要观点

  1. 迟分算法结合 `jina-embeddings-v3` 模型能有效提取长文本中最相关的片段。 该方法利用一维卷积方式，在保证上下文连贯性的前提下，选择与查询最相关的文本块，提高检索效率。
  2. 综合多种因素进行 URL 重排能提升 LLM 选择 URL 的准确性。 通过考虑频率信号、路径结构、语义相关性、最后更新时间等因素，对 URL 进行排序，引导 LLM 优先选择包含答案的 URL。
  3. 两级分化的搜索架构变得越来越流行。 轻量级 SQL/BM25 负责检索输入，LLM 负责检索输出，中间层模型价值转移到上下文窗口内的过滤、去重、排序等任务。

💬 文章金句

  • 在 DeepSearch 中，Embeddings 模型仅适用于诸如 STS（语义文本相似度）任务之类的查询去重。
  • 我们非常 80-20，非常务实，务实到我们只关心搜索系统的最最本质的需求。
  • 你可以把这个问题看作 '上下文内的 PageRank'，只不过，我们是在一次会话中对数百个 URL 打分。
