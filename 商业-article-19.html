Asahi Linux 首席开发者发辞职信引热议：Rust 和 C 的混合堪比毒瘤癌症

2/8/2025, 1:57:25 AM

📌 一句话摘要

Asahi Linux 首席开发者因 Rust 与 C 混合在 Linux 内核中引发的争议而辞职，凸显了多语言融合的挑战。

📝 详细摘要

Asahi Linux 首席开发者赫克托 · 马丁因对 Linux 内核开发流程和社群管理方式失去信心而辞职，事件起因于 Linux 内核维护者对在内核 DMA API 中使用 Rust 代码的抵制，他们认为多语言混合会降低内核的可维护性。围绕 DMA API 抽象层维护的权责划分问题，社区展开激烈争议。该事件不仅对 Asahi Linux 项目造成潜在影响，更引发了对 Linux 内核开发模式的讨论，Rust for Linux 项目的未来走向充满不确定性。

💡 主要观点

  1. Rust 代码集成到 Linux 内核 DMA API 存在技术争议，核心在于抽象层维护的权责划分 内核维护者希望非 C 驱动程序有自己的私有 C 代码绑定，不希望由 Rust for Linux 项目单独维护 DMA Rust 抽象层，这引发了关于代码可维护性和责任归属的争议。
  2. Linus Torvalds 的态度决定 Rust for Linux 项目的命运 如果 Linus Torvalds 接受 Rust 补丁，Rust for Linux 项目还有机会，否则项目可能会失败。Linus 的决策将对 Linux 内核的未来发展方向产生重要影响。
  3. Rust 旨在解决 C 和 C++ 的内存安全问题 Rust 语言在内存安全方面具有优势，可以避免 C 和 C++ 代码中常见的内存安全漏洞，这使得 Rust 在安全性要求高的场景下具有吸引力。
  4. 社群管理方式和流程对开发者积极性的影响 赫克托 · 马丁的辞职声明表明，内核开发流程和社群管理方式对开发者的积极性和参与度有重要影响，不合理的流程和管理可能导致开发者失去信心。
