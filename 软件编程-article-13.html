📌 一句话摘要

Node.js v22.14.0 LTS 版本发布，提供长期支持，并更新和修复了密码学模块（crypto）、文件系统模块（fs）和模块（module）等。

📝 详细摘要

Node.js v22.14.0 'Jod' (LTS) 版本现已发布，包含多个模块的更新和修复。密码学模块（crypto）包括根证书更新和 checkPrime 修复，增强了安全性。文件系统模块（fs）引入了 glob 模式的 `exclude` 选项，提高了文件操作的灵活性。模块（module）添加了 `ERR_UNSUPPORTED_TYPESCRIPT_SYNTAX` 和用于查找 `package.json` 文件的 `findPackageJSON` 实用程序，简化了模块解析。进程模块（process）引入了 `process.ref()` 和 `process.unref()` 方法。此外，测试运行器模块（test_runner）通过新的 API 增强了测试功能。此版本还包含依赖项更新、文档改进和错误修复，有助于 Node.js 运行时的稳定性和可靠性。

💡 主要观点

  1. 密码学模块（crypto）更新根证书并修复 checkPrime，增强了安全性和可靠性。 将根证书更新到 NSS 3.107，并修复了处理大型缓冲区时 checkPrime 崩溃的问题，从而提高了加密操作的安全性和稳定性。
  2. 文件系统模块（fs）增加了对 glob 模式中 exclude 选项的支持，从而提高了文件操作的灵活性。 在 glob 模式中引入 `exclude` 选项，允许用户排除特定文件或目录，从而提供更自定义的文件搜索和处理功能。
  3. 模块（module）增强了 TypeScript 支持，并提供了用于模块解析的实用程序，从而提高了开发人员的效率。 添加 `ERR_UNSUPPORTED_TYPESCRIPT_SYNTAX` 和 `findPackageJSON` 实用程序，简化了在 Node.js 开发中使用 TypeScript 并简化了模块解析过程。
  4. 测试运行器模块（test_runner）引入了新的 API，增强了测试功能。 添加 `TestContext.prototype.waitFor()`、`t.assert.fileSnapshot()` 和 `assert.register()` API 丰富了测试框架的灵活性和功能。
  5. assert 模块提高了 myers_diff 函数的性能。 assert 模块中 `myers_diff` 函数的性能改进有助于在测试和调试期间进行更高效、更快速的比较。

💬 文章金句

  • 密码学模块（crypto）：将根证书更新到 NSS 3.107
  • 文件系统模块（fs）：允许 globs 中的 `exclude` 选项接受 glob 模式
  • 模块（module）：添加 ERR_UNSUPPORTED_TYPESCRIPT_SYNTAX
  • 模块（module）：添加 `findPackageJSON` 实用程序
  • 测试运行器模块（test_runner）：添加 TestContext.prototype.waitFor()
  • assert：使 myers_diff 函数更高效
