Rust 补丁两年零通过，34 年开发者煽动网友逼 Linus 表态，维护者激烈反击：Rust 就是 Linux 的“癌症”！

📌 一句话摘要

Linux 内核社区因 Rust 语言引入问题爆发争论,开发者 Martin 因补丁未通过愤然离职,暴露了社区在技术选型和开发流程上的分歧。

📝 详细摘要

文章报道了 Linux 内核社区关于是否在内核中使用 Rust 语言的争论。由于 Rust 代码补丁长期未被审核通过,Asahi Linux 首席开发者 Hector Martin 愤然离职。事件起因是 Martin 提交的允许 Rust 编写的设备驱动程序调用内核核心 DMA API 的提议,遭到了内核维护者 Christoph Hellwig 的强烈抵制,Hellwig 认为引入多种语言会增加复杂性和安全风险。Rust 旨在提高内核的安全性,避免 C 和 C++ 代码中常见的内存安全漏洞。Linus Torvalds 也加入争论,批评 Martin 在社交媒体上争取支持的行为,加剧了社区内部的对立情绪。这场争论暴露了 Linux 社区在技术选型、维护成本和开发流程上的分歧,对 Rust for Linux 项目的未来发展产生了不确定影响。

💡 主要观点

  1. Rust 引入 Linux 内核面临维护者阻力 内核维护者 Christoph Hellwig 担心增加内核复杂性和安全风险,坚持 C 语言,对 Rust 持谨慎态度,阻碍了 Rust 代码的合并。
  2. Linus 对 Rust 态度不明,引发社区信任危机 Linus 未明确支持或反对 Rust,对 Martin 在社交媒体的行为表示不满,加剧了社区内部的对立情绪。Linus 的态度对事件走向有决定性影响。
  3. 贡献者提交代码长期未被审核 Martin 提交的补丁系列两年内未被审核,缺乏有意义的反馈和交流,对内核开发流程和社区管理方式失去信心。

💬 文章金句
