📌 一句话摘要

Roc 编程语言因 Rust 编译速度慢，决定用 Zig 重写编译器，旨在提高开发效率、优化工具链并提升整体性能。

📝 详细摘要

Roc 编程语言，作为一种旨在简化应用程序开发的语言，其作者决定用 Zig 语言重写编译器，主要原因是 Rust 编译速度较慢，严重影响了开发效率和体验。现有 Rust 代码库在解析器的容错性、格式化器的行宽限制等方面存在问题和改进需求。文章详细阐述了重写的原因，包括规范化、文档生成、单态化和 LLVM 代码生成等方面的问题。此外，文章还对比了 Rust 和 Zig 在内存安全、内存管理、生态系统和工具链等方面的优缺点，最终认为 Zig 更适合 Roc 编译器的需求，且预期重写后能带来性能提升。此次重写目标是发布 Roc 0.1.0 版本，并提高编译器的可靠性和正确性，为未来的性能优化打下基础。

💡 主要观点

  1. Rust 编译速度慢是 Roc 转向 Zig 的主要原因 Rust 编译速度严重影响了开发效率和工作体验，而 Zig 在编译速度方面具有优势，能够提供更快的反馈循环。
  2. Zig 的工具链更适合 Roc 编译器的需求 Zig 易于使用 musl libc 编译静态链接的 Linux 二进制文件，支持直接生成 LLVM 位码，这些特性 Roc 团队非常需要。
  3. Zig 的内存管理和语言特性更适合 Roc 编译器的需求 Zig 围绕传递分配器进行内存管理，并内建了许多优化结构如 MultiArrayList，简化了结构体数组的编程。
  4. 重写 Roc 编译器旨在提高可靠性和正确性，并为未来的性能优化打下基础 通过尽早进行模糊测试、边做边记录和改进错误处理等方式，Roc 团队希望构建一个更可靠、可用的编译器实现。
