Go 1.24 扩展了对 WebAssembly (Wasm) 的支持

2/14/2025, 12:00:00 AM

📌 一句话摘要

Go 1.24 通过增强的 WebAssembly (Wasm) 支持，提升了 Go 在高性能边缘计算应用中的性能和灵活性。

📝 详细摘要

Go 1.24 显著增强了对 WebAssembly (Wasm) 的支持，引入了 `go:wasmexport` 编译器指令，使得 Go 函数能够导出至 Wasm，并以接近原生性能的速度执行。同时，新的 WASI reactor 构建标志支持 Go 应用在初始化后保持运行状态，适用于长时间运行的服务。Google Cloud 现在支持在边缘运行 Go 编译的 Wasm 插件，通过 Service Extensions 和 Application Load Balancers 实现。

💡 主要观点

  1. `go:wasmexport` 编译器指令使 Go 函数可以被 Wasm 主机调用。 这一特性使得 Go 应用程序能够像插件一样被 Wasm 主机使用，扩展了 Go 在 Wasm 环境中的应用场景。
  2. WASI reactor 构建标志支持长时间运行的 Wasm 应用。 通过保持应用在初始化后持续运行，确保导出的函数在无需重新初始化的情况下保持可调用，适用于长期运行的服务。
  3. Google Cloud 通过 Service Extensions 和 Application Load Balancers 支持在边缘运行 Go 编译的 Wasm 插件，实现低延迟和业务定制化。 利用 Service Extensions 和 Application Load Balancers，可以在 Google Cloud 边缘环境中运行 Go 编译的 Wasm 插件，降低延迟，提升应用响应速度，并满足多样化的业务需求。

💬 文章金句

  • Go 1.24 also significantly expands its capabilities for WebAssembly (Wasm), a binary instruction format that provides for the execution of high-performance, low-level code at speeds approaching native performance.
  • With a new `go:wasmexport` compiler directive and the ability to build a reactor for the WebAssembly System Interface (WASI), developers can now export functions from their Go code to Wasm --- including in long-running applications --- fostering deeper integrations with Wasm hosts and unlocking new possibilities for Go-based Wasm applications.
  • Go 1.24 的 `go:wasmexport` 指令和 WASI reactor 构建能力，为开发者提供了构建高性能、长时间运行的 Wasm 应用的新途径
