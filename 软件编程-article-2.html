JetBrains 取消在 Fleet 上构建 Kotlin Multiplatform IDE 计划

2/22/2025, 6:40:28 AM

📌 一句话摘要

JetBrains 取消在 Fleet 上构建 Kotlin Multiplatform IDE 的计划，转而改进 IntelliJ，引发开发者对 Fleet 未来的担忧。

📝 详细摘要

JetBrains 放弃了在 Fleet 上构建 Kotlin Multiplatform (KMP) IDE 的计划，转而在 IntelliJ IDE 中改进 KMP 功能，此举引发了开发者对 Fleet 未来发展的质疑，同时也影响了 KMP 开发者和整个 Kotlin 生态。JetBrains 解释称，这一转变是由于客户反馈更希望在 IntelliJ 等现有 IDE 中获得 KMP 支持。尽管 Fleet 在不断改进，但其采用率仍然有限，有开发者认为 Fleet 更多的是一项实时协作技术的试验，而非未来的主要 IDE 平台。谷歌对 KMP 的官方支持也倾向于 Android Studio，而非其他 IDE。

💡 主要观点

  1. JetBrains 放弃 Fleet 上构建 KMP IDE 计划，转向 IntelliJ。 JetBrains 听取用户反馈，决定在更成熟的 IntelliJ IDE 上改进 KMP 支持，而非投入资源在 Fleet 上构建独立 IDE。
  2. 开发者对 Fleet 的未来表示担忧。 Fleet 作为 IntelliJ 的替代品，推出后采用率有限，此次 KMP 计划的取消加剧了开发者对其发展前景的疑虑。
  3. Fleet 更多的是一项实时协作技术的试验。 有开发者和业内人士认为，Fleet 的重点在于实时协作技术，可能不会成为 JetBrains 的主要 IDE 平台。
  4. 谷歌官方支持 KMP，倾向于 Android Studio。 谷歌在 I/O 大会上宣布对 KMP 的官方支持，并更希望在 Android Studio 中支持 KMP，这进一步影响了 JetBrains 的决策。
