Curl 之父：我是如何枕着 18 万行 C 代码还能安稳入睡的

3/2/2025, 2:26:28 AM

📌 一句话摘要

Curl 作者分享了在 C 语言环境下，通过严格规范、工具预防、海量测试和高效修复等手段，保障这个拥有 200 亿次安装量的开源项目的安全性。

📝 详细摘要

本文主要介绍了 Curl 作者 Daniel Stenberg 在 FOSDEM 开源大会上分享的保障 Curl 项目安全性的经验。面对 Curl 包含 18 万行 C 代码的现状，Stenberg 明确表示不会用 Rust 等其他语言重写，而是通过持续迭代和打磨来提升安全性， 保证现有 C 代码库的稳定和安全。 他同时强调，对 Rust 等新技术持开放态度，允许第三方依赖项使用 Rust 编写。文章还介绍了 Curl 的代码规范，禁止使用如 `gets()`、`scanf()` 等容易出错的函数，并通过“折磨测试”来测试内存分配失败时的处理情况，以及多层次测试方法、漏洞修复流程和漏洞赏金计划等， 共同确保了 Curl 在全球 200 亿次安装中的可靠性。

💡 主要观点

  1. 严格的代码规范是保障 C 语言项目安全性的基础 通过禁止使用容易出错的函数，例如 `gets()`、`scanf()`、`strcpy()` 和 `sprintf()` 等，并使用工具进行检查，可以有效避免潜在的缓冲区溢出等安全问题。
  2. 多层次的测试体系是发现和修复漏洞的关键 包括单元测试、库测试、工具测试以及静态和动态分析器等，可以全面覆盖代码的各个方面，及时发现潜在的安全风险。
  3. 高效的漏洞修复流程和漏洞赏金计划能提升安全性 与安全平台合作，鼓励安全研究人员提交漏洞报告，并及时修复和公开漏洞信息，有助于提升项目的整体安全性。

💬 文章金句

  • 不管你经验有多丰富------C 标准库中的一些函数真的不建议在代码中使用。我们用工具检查并禁止使用它们，这样就不会不小心让它们混进去了。
  • 我们不会用 Rust 重写 Curl，也不会用任何其他语言------我们不会重写它。它已经在那里了。
  • 实际上，目前我们收到的有效报告和'AI 垃圾报告'的比例差不多
