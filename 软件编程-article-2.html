Visual Studio 推出 GitHub Copilot 驱动的本地代码评审功能，在提交前及早发现问题，提升代码质量。

📝 详细摘要

本文介绍一项新的 Visual Studio 功能，该功能利用 GitHub Copilot 在 Git 提交前审查本地代码变更。这有助于开发人员在开发过程中及早发现潜在的性能和安全问题，最终节省时间并提高代码质量。要开始使用，用户需要有效的 GitHub Copilot 订阅，并且需要在 Visual Studio 设置中启用特定的功能标志。“Git 更改”窗口中的“代码评审”按钮会触发 Copilot 审查代码更改并提供内联建议。鼓励用户审查 Copilot 建议的准确性，并通过调查提供反馈，以进一步改进该功能。

💡 主要观点

  1. 及早代码评审，防患未然 由 GitHub Copilot 驱动的新的 Visual Studio 功能允许开发人员在提交之前在本地审查代码更改，从而及早发现潜在问题，如性能瓶颈或安全漏洞。
  2. GitHub Copilot 集成简化了代码评审过程。 通过将 GitHub Copilot 集成到 Visual Studio 的 Git 工作流程中，开发人员可以直接在其开发环境中轻松访问 AI 驱动的代码建议和改进。
