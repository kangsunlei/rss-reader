Github 超 2 万星，OpenManus 核心作者聊 Agent 发展趋势

3/10/2025, 1:16:05 PM

📌 一句话摘要

OpenManus 核心作者分享 Agent 发展趋势，重点介绍了其 3 小时快速复刻 Manus 的技术实现和未来发展方向。

📝 详细摘要

文章是对 OpenManus 团队核心成员的一次分享内容的整理，主要讨论了 Agent 技术的发展趋势以及 OpenManus 的技术实现原理。基于 MetaGPT 团队的技术积累，OpenManus 团队在短时间内复刻了 Manus，并在 Github 上获得了大量关注。分享中，团队成员对大模型能力提升、Agent 规划、工具使用、Memory 管理、商业化落地等多个方面进行了深入分析，并提出了未来 Agent 技术的发展方向，包括规划能力增强、标准化评测和模型适配等。此外，文章还介绍了 MetaGPT 团队在多智能体研发方面的技术积累，并探讨了 Agent 在代码生成领域的商业化前景。

💡 主要观点

  1. Agent 规划方面的进步依赖于模型能力提升和外部结构辅助 Agent 的规划能力提升不仅取决于大模型本身的进步，还需要借助外部结构的辅助，例如引入更复杂的结构进行辅助规划。
  2. Agent 工具使用的难点在于工具选择和上下文信息处理 随着工具数量增加，Agent 需要准确选择最合适的工具，并避免因工具参数定义不合理或上下文信息混淆而导致错误。
  3. Memory 管理涉及成本和效率问题，可通过多智能体规划和任务分解来优化 直接使用完整 memory 会增加处理时间和成本，可采用多智能体规划和任务分解的方式，例如通过规划工具生成规划，每个任务间的 memory 不完全共享。

💬 文章金句

  • 随着大模型能力的增加，许多问题的解决成功率会提高，但问题本身并不会消失。
  • Agent 与大模型可能是一种垂直或正交的关系。框架本身的提升会因为模型能力的提升而带来更多功能，它们并不冲突。
  • Agent 商业化的重要比拼在于将真实场景中的任务和效果，包括个性化的功能，做到极致，用户才会持续使用 Agent。
  • 未来很可能通过集成多个小模型的能力，实现完整甚至超过大模型的效果，并在推理速度、token 消耗和费用上明显降低成本。
  • 在真正有效解决用户实际需求方面，多智能体的商业前景是明确且强烈的，也是 Agent 技术目前能较好解决的场景。
  • MetaGPT 框架沉淀了多智能体协作的核心思想
