💡 主要观点

  1. LLM 是高级自动补全工具，用于增强能力，需设定合理的期望。 LLM 的核心能力是预测 token，能按正确顺序拼接 token，但会犯错。应将其视为结对编程助手，快速查找信息和执行任务。
  2. 管理上下文是使用 LLM 的关键，需注意模型的训练截止日期。 上下文包括提示词和对话记录，要了解模型所熟悉的库的版本，并手动提供最新示例，以保证 LLM 能给出有效的回答。
  3. 明确告诉 LLM 该做什么，并进行测试，将 LLM 作为对话伙伴。 在编写代码时，要像对待数字实习生一样，给出详细指令。同时，必须手动测试 LLM 生成的代码，并不断修改，直到满意为止。
  4. 利用 Vibe-Coding 学习 LLM，并使用支持运行代码的工具。 通过不断尝试和疯狂 Vibe-Coding，快速建立直觉，了解 LLM 的优缺点。同时，选择支持安全运行代码和自动优化的工具。
  5. 利用 LLM 解决实际代码问题，并加速学习新技术。 LLM 能加速实现想法，交付更多项目，并学习新事物。此外，LLM 擅长回答代码问题，可用于探索新代码库。

💬 文章金句

  • LLM 本质上仍然只是一个 '高级自动补全' 工具。
  • 用它们来增强你的能力。我目前最喜欢的思维模型是把它们想象成一个过度自信的结对编程助手，它查找信息的速度快如闪电，能即刻给出相关示例，还能毫无怨言地执行枯燥的任务。
  • 想要用好 LLM，最核心的技巧就是管理上下文 ------ 也就是当前对话中提供给模型的文本信息。
  • LLMs 无法替代人类的直觉和经验。我花了很多时间与 GitHub Actions 打交道，我知道该寻找什么样的东西，在这种情况下，我更快地介入完成项目，而不是继续用提示来尝试达到目标。
  • 这就是我如此重视 LLMs 给我带来的生产力提升的原因：这并非只是关于更快地完成工作，而是关于能够交付那些我原本根本无法证明投入时间是值得的项目。
  • 准备好接手：LLMs 无法替代人类的直觉和经验。
