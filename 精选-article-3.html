OpenAI 发布智能体开发套件：AI 能自主操作计算机，但建议用户“监工”

3/12/2025, 12:42:02 AM

📌 一句话摘要

OpenAI 发布 Agent 开发套件，旨在通过简化 AI 智能体的构建和应用，推动 AI 自主操作计算机的发展。

📝 详细摘要

OpenAI 发布了一系列针对 AI Agent 的开发工具与 API，核心是 Responses API，它融合了对话式 API 的简洁性和助手 API 的工具调用能力，降低了复杂业务流程的架构复杂度。内置工具包括网络搜索、文件搜索和计算机使用。同时发布了智能体 SDK，用于协调单代理和多代理工作流，以及集成的可观测性工具。Responses API 支持网络搜索、文件搜索和计算机使用等功能，旨在将模型与现实世界连接。OpenAI 还强调了数据安全和操作安全，并建议人工监督，同时默认不会将企业数据用于模型训练。未来 OpenAI 将弃用 Assistants API，目标时间为 2026 年年中。智能体 SDK 显著简化多智能体工作流管理，包含智能体配置、流程交接、安全护栏、追踪与可观测性等改进。

💡 主要观点

  1. Responses API 是 OpenAI 智能体开发的核心，融合对话式和助手 API，简化智能体开发 Responses API 结合了对话式接口的易用性与助手 API 的工具调度能力，降低了开发复杂智能体的门槛，并提供更灵活的工具调用方式。
  2. 内置工具如网络搜索和文件搜索，增强智能体实用性 通过集成网络搜索和文件搜索等工具，智能体可以更方便地获取外部信息，从而在各种实际应用场景中发挥更大的作用。
  3. 智能体 SDK 简化多智能体工作流管理 智能体 SDK 提供智能体配置、流程交接、安全护栏和可观测性等功能，简化了多智能体系统的构建、管理和优化。

💬 文章金句

  • OpenAI 发布针对 AI Agent 打造的系列工具与 API，助力开发者更便捷地创建可自动执行任务的 AI Agent。
  • Responses API 深度融合对话式接口的易用性与助手 API 的工具调度能力。
  • OpenAI 默认不会将存储于其平台的企业数据用于模型训练，该承诺在数据处理协议中有明确条款约束。
  • 建议涉及系统操作时保持人工监督。
  • 智能体 SDK 适用于多种现实场景，包括客户支持自动化、多步骤研究、内容生成、代码审查与销售线索挖掘。
  • OpenAI 未来将弃用 Assistants API，目标时间为 2026 年年中
