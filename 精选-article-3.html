📝 详细摘要

文章以“十个关键词”为线索，系统地解析了 DeepSeek 大模型的技术特点及其在推理能力和训练成本方面的突破。首先介绍了多头潜在注意力（MLA）机制，它通过低秩联合压缩技术有效降低了推理成本。其次，阐述了专家混合模型（MOE）在 DeepSeek 中的应用，包括动态路由和专家共享机制。然后，深入分析了 DeepSeek 在负载均衡、模型蒸馏、纯强化学习、GRPO 算法、过程奖励模型等方面的创新实践。此外，文章还探讨了 DeepSeek 通过 PTX 汇编绕开 CUDA，以及使用 DPU 替代 NVLink 对英伟达生态的潜在冲击。最后，借用杰文斯悖论解释了 DeepSeek 降低推理成本反而可能增加 GPU 总需求的现象。

💡 主要观点

  1. MLA 机制有效降低了 DeepSeek 的推理成本 通过低秩联合压缩技术，将多个输入向量压缩为一个隐藏向量，减少了键值缓存，提高了模型预测效率。
  2. DeepSeek 通过纯强化学习降低训练消耗 抛弃了传统的监督微调范式，直接进行强化学习，依靠简单的奖惩信号对模型进行优化，大幅降低训练成本。
  3. GRPO 算法是 DeepSeek 实现高效策略优化的关键 该算法只考虑群体之间的相对关系和影响，无需价值模型和奖励模型，降低了训练成本，通过规则直接评估回答的准确性。
  4. PTX 汇编直接操作硬件层，可以获得更高的计算效率 DeepSeek 通过 PTX 汇编，可以独立于具体的 GPU 硬件架构，直接操作硬件层，从而获得更高的计算效率。

💬 文章金句

  • DeepSeek 则完全抛开了利用大量思维链 COT 模板进行监督微调的范式，直接进行强化学习，依靠简单的奖惩信号对模型进行优化，大幅降低训练消耗。
  • 通过 PTX（Parallel Thread Execution，可以独立于具体的 GPU 硬件架构，介于高级编程语言和底层 GPU 机器码之间）汇编直接操作硬件层，可以获得更高的计算效率。
  • DeepSeek 带来了推理成本的大幅下降，大家第一反应会认为这会减少 GPU 的算力消耗。但由于推理效率提高，AI 应用场景会迅速扩张，甚至以前没有能力大量采购 GPU 的中小企业，也能参与这场 AI 盛宴，最终导致 GPU 总需求增加。
  • DeepSeek 向世人证明了，即使不需要非常高端的 GPU，也能进行高效的模型推理。
