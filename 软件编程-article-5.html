Node.js 23.6 现在原生支持 TypeScript

3/9/2025, 12:00:00 AM

📌 一句话摘要

Node.js 23.6 原生支持 TypeScript，允许开发者直接运行 .ts 文件并简化开发，但存在对 tsconfig.json 等配置文件的支持限制。

📝 详细摘要

Node.js 23.6 引入了原生 TypeScript 支持，使开发者能够直接运行 .ts 文件而无需额外的配置，从而简化了开发过程。该功能通过移除类型注解的方式实现，但存在诸如不支持 tsconfig.json 和某些 TypeScript 语法等限制。通过支持“可擦除语法”，Node.js 更加接近 Deno 和 Bun 这样的 TypeScript 原生运行时，减少了 Node.js 开发者的构建步骤。虽然 Deno 提供了额外的功能，但 Node.js 的改进是朝着更现代的开发体验迈出的一步。开发者应查阅官方文档以了解支持的语法和限制。

💡 主要观点

  1. Node.js 23.6 原生支持 TypeScript，简化了开发过程。 开发者无需额外的编译步骤和配置，可以直接运行 .ts 文件，从而降低入门门槛并提高开发效率。
  2. 原生 TypeScript 支持存在限制，不支持所有 TypeScript 语法。 Node.js 仅支持“可擦除语法”，不支持需要 JavaScript 代码生成的语法或 tsconfig.json 文件。开发者需要注意兼容性。
  3. Node.js 原生 TypeScript 支持是朝着更现代的开发体验迈出的一步，但仍需要不断改进。 与 Deno 等运行时相比，Node.js 在 tsconfig.json 支持和 TypeScript 分发方面仍有差距，未来有待改进。

💬 文章金句

  • Node 开发者现在可以运行 TypeScript 文件，而无需任何额外的配置或标志。
  • Node.js 忽略 `tsconfig.json` 文件，因此明确表示不支持依赖于 `tsconfig.json` 中设置的功能。
  • 受支持的 TypeScript 语法集被称为可擦除语法。
  • 原生 TypeScript 支持使 Node 更加接近 Deno 或 Bun 等 TypeScript 原生运行时。
