VisualStudio.Extensibility：管理 .NET 运行时版本

2/10/2025, 4:17:53 PM

📌 一句话摘要

介绍 VisualStudio.Extensibility SDK 如何通过平滑过渡 .NET 运行时版本，提升扩展的性能和可靠性，并最大限度地减少对用户的影响。

📝 详细摘要

本文主要介绍了 VisualStudio.Extensibility SDK 在管理 .NET 运行时版本方面的改进。该 SDK 允许开发者创建在 Visual Studio IDE 进程之外运行的扩展，从而提高性能和可靠性，并且无需重启 Visual Studio 即可安装。文章重点介绍了 Visual Studio 2022 (17.14 Preview 1) 中引入的 .NET 运行时版本管理功能。该功能为扩展用户提供了平滑的 .NET 运行时版本更新体验，并为扩展开发者提供了在新运行时版本可用时进行调试和兼容性测试的机会。此外，文章还讨论了 .NET 运行时版本更新带来的潜在问题，以及在主 devenv.exe 进程中运行扩展的替代方案，并提供了通过 GitHub 仓库和开发者社区提供反馈的渠道。

💡 主要观点

  1. VisualStudio.Extensibility 扩展可在进程外运行，提升性能和可靠性。 通过在 Visual Studio IDE 进程之外运行扩展，可以避免对主进程的影响，从而提高整体性能和稳定性。
  2. Visual Studio 将平滑过渡 .NET 运行时版本，对用户体验的影响较小。 即使扩展未指定支持最新的 .NET 运行时版本，Visual Studio 也会继续加载这些扩展，保证用户工作流程不受重大影响。
  3. 扩展开发者可以使用新的 F5 调试体验在新运行时版本发布前进行兼容性测试，降低迁移成本。 新的 F5 调试体验允许开发者在新的 .NET 运行时版本中加载和测试他们的扩展，以便及早发现和解决兼容性问题，降低迁移成本。

💬 文章金句

  • VisualStudio.Extensibility 的主要优势在于，它允许扩展在 .NET 8 上以进程外模式运行，从而提供更现代、更高性能的运行时环境。
  • 当 .NET 运行时版本更新时，扩展用户不应预期其工作流程会发生重大或侵入性更改。
  • 我们理解，即使 Visual Studio 2022 停止积极开发，这种更新运行时的模型也与通常在进程内 VSSDK 模型中维护扩展的方式不同。
