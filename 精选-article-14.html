万字长文详解 DeepSeek 核心技术

📌 一句话摘要

本文深入解读 DeepSeek 系列模型的技术创新，重点介绍了其在 MoE 架构、优化算法以及 R1 系列模型上如何通过强化学习实现卓越推理能力的突破。

📝 详细摘要

本文详细介绍了 DeepSeek 系列模型的技术创新，包括 DeepSeek MoE 架构的细粒度专家划分、共享专家分离，以及为解决负载不均衡问题而提出的负载均衡策略。深入解析了 GRPO 算法相对于 PPO 的改进，通过减少 Value 模型来降低计算资源消耗。阐述了 MLA 如何通过低秩分解减少 KV Cache，降低推理成本。介绍了 MTP 如何通过一次性预测多个 token 来提升训练效率和推理速度。特别强调了 R1-Zero 完全基于强化学习训练出推理模型的突破性意义，以及 DeepSeek V3 在训练效率和成本控制方面的优势。同时，突出了 R1 在 R1-Zero 基础上，通过多阶段训练策略解决可读性和语言混杂问题，达到工程化落地的效果。整体而言，文章对 DeepSeek 的核心技术进行了全面而深入的解读。

💡 主要观点

  1. DeepSeek MoE 架构通过细粒度划分和专家分离提升模型性能 细粒度划分降低专家参数量，专家分离使模型能同时捕捉共性和差异性，动态调整路由倾向，从而提升整体效果。
  2. GRPO 算法优化 PPO，减少计算资源消耗 GRPO 通过对多个预测结果评分取平均来替代 Value 模型，减少训练过程中的前向和反向传播计算，降低资源消耗。
  3. MLA 通过低秩分解减少 KV Cache，降低推理成本 MLA 对 KV Cache 作低秩分解，得到低维隐向量，推理时只需缓存隐向量，从而减少数据量，降低推理成本。
  4. DeepSeek 通过强化学习在推理能力上取得重大突破，R1-Zero 验证了完全基于强化学习训练推理模型的可行性 R1-Zero 通过基于规则的 Reward 模型和 GRPO，实现了自我进化，无需 SFT 训练即可在推理任务中表现出色，为以 RL 为中心的推理 AI 进一步铺平了道路。
  5. DeepSeek R1 通过多阶段训练策略达到 OpenAI-o1 水平 R1 通过冷启动、推理导向 RL、监督微调和全场景 RL 等多阶段训练，在 R1 Zero 基础上进一步提升模型效果，并解决可读性和语言混杂问题。
  6. DeepSeek V3 通过混合精度框架和 MTP 等技术，显著提升训练效率并降低成本 DeepSeek V3 模型使用了 FP8 训练，并提出了一种精细的量化策略，另外为了进一步减少 MoE 训练中的内存和通信开销，作者在 FP8 中缓存和分发激活值，同时以 BF16 格式存储低精度优化器状态。

💬 文章金句

  • DeepSeek-R1-Zero 是第一个公开的研究，验证了 LLM 的推理能力可以完全通过强化学习（Reinforcement Learning，RL）来激励，而不需要 SFT。
  • DeepSeek 对社区的贡献还在于，他们提供的 API 展示了模型的思考过程，让从业人员以及使用者能够全方位地了解到 Long-CoT 的特点与作用。
  • DeepSeek-R1-Zero 展示出了自我进化能力，随着强化学习训练进程的深入，模型的思考时间在增加，并自发出现了诸如反思，模型重新审视和重新评估其先前步骤以及探索解决问题的替代方法等更加复杂的行为。
  • DeepSeek 通过多阶段训练策略，在 R1 Zero 基础上实现了模型效果和易用性的平衡，为实际应用奠定了基础。
