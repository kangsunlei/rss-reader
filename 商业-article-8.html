“我真的受够了 Ubuntu！”

2/8/2025, 9:14:27 AM

📌 一句话摘要

作者因 Ubuntu 的升级问题和 Snap 包管理的缺陷感到强烈不满，转而推荐 Fedora。

📝 详细摘要

本文作者分享了自己使用 Ubuntu 操作系统遇到的问题，包括系统升级时频繁出现故障，以及对 Snap 包管理机制的强烈不满。作者认为 Ubuntu 的升级过程不够稳定，Snap 包的强制自动更新和兼容性问题影响了用户体验和工作效率。同时，作者对比了 Flatpak 的实现方式，认为其更为合理，用户可以自主选择更新。最后，作者推荐 Fedora 作为更适合工作的 Linux 发行版，并希望 Canonical 能够重新审视 Ubuntu 的构建方法。

💡 主要观点

  1. Snap 包管理机制存在缺陷，强制更新和兼容性问题突出 作者对 Ubuntu 强制使用 Snap 包管理机制表示不满，认为其自动更新和兼容性问题影响了用户对系统的控制，并可能导致工作效率下降。
  2. Ubuntu 的升级过程经常出现问题，影响用户体验 作者及其家人朋友在使用 Ubuntu 升级时遇到各种问题，包括系统崩溃、界面卡顿、快捷方式失效等，严重影响了系统的可用性。
  3. Flatpak 是 Snap 包管理的一种更合理的实现方式 Flatpak 允许用户选择更新，避免了 Snap 包强制自动更新带来的问题，在实现上更为灵活和用户友好。
  4. Fedora 是 Ubuntu 的一个不错的替代选择 作者认为 Fedora 运行良好，适合完成工作，只要 IBM 不干预，它会是一个非常适合工作的 Linux 发行版。

💬 文章金句

  • 我真的是很讨厌 Snaps。理论上来说，Snaps 听起来很棒，但 Canonical 的糟糕实现和强行推广却把它搞得一团糟。
  • 将 Ubuntu 从 20.04 升级到 22.04 的过程中，系统彻底崩溃了
  • 每次升级，都会出现一些问题。
  • 其实，Snaps 是可以做得更好的，Flatpak 就是一个例子。
  • Fedora 运行得很好。虽然它并不完美，但我很喜欢它。
  • 希望 Canonical 能重新审视他们构建 Linux 发行版的方法。
