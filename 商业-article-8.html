Manus 爆火启示录：套壳没有护城河

3/7/2025, 8:04:34 AM

📌 一句话摘要

Manus 通过套壳现有模型快速推出 AI Agent 产品，但缺乏自主技术和长期竞争力。

📝 详细摘要

文章分析了近期爆火的 AI Agent 产品 Manus，该产品定位为通用 AI Agent，旨在通过多模型和智能体实现自主完成复杂任务。文章指出 Manus 本质上是一个“套壳”产品，集成了 Claude 和 Qwen 等底层模型，并通过工程化手段将 Agent 框架、搜索引擎和各种工具组合在一起。文章肯定了 Manus 在用户体验方面的优势，特别是“所见即所得”的交付逻辑，但也指出其依赖外部模型、缺乏技术壁垒，以及高昂的成本和易被复制是其潜在的风险。此外，文章还讨论了 Agent 应用赛道的发展趋势和商业模式，对比了订阅收费和任务导向收费的模式，并分析了 Manus 与 OpenAI 等公司的竞争态势。文章强调了 Agent 能力提升最终依赖于大模型能力的提升，以及端到端模型训练的重要性。

💡 主要观点

  1. Manus 通过集成现有模型和工具快速实现 Agent 功能。 Manus 没有自研大模型，而是通过集成 Claude 和 Qwen 等模型，并结合 Agent 框架、搜索引擎等工具，实现了初步的 Agent 功能。
  2. “套壳”模式使 Manus 缺乏核心技术壁垒。 Manus 的模式易于被复制，开源社区已快速复现其核心工作流，长期来看，缺乏自主技术难以形成竞争优势。
  3. 成本控制和用户体验是 Manus 短期内脱颖而出的关键。 在长期发展受限的情况下，Manus 需要通过优化成本结构和提升用户体验，才能在短期内获得市场认可。
  4. Manus 的各种花式营销操作，无非是想要抢先押注 Agent 应用赛道。 Manus 通过营销手段，希望在 Agent 应用领域占据先机，从而在大模型时代的供需关系转变中获得优势。

💬 文章金句

  • Manus 官方定位为 "通用 AI Agent"，即一个多模型、多智能体产品，拥有 "独立思考" 能力，能够对复杂的通用任务进行多步骤拆解并执行。
  • Manus 直言不讳地表示其产品属于 "套壳"，底层模型采用了 Claude 和 Qwen，单次任务消耗的成本大概在两美元左右。
  • Agent 能力提升很大程度还是要依赖于大模型能力。
  • 众所周知，"套壳" 产品没有护城河。
