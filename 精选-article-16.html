DeepSearch 与 DeepResearch 的设计和实现

2/26/2025, 8:41:23 AM

📌 一句话摘要

本文深入探讨 DeepSearch 和 DeepResearch 的设计与实现，强调关键技术要素，并分享 Jina AI 在构建相关系统中的深度实践与经验总结。

📝 详细摘要

文章详细介绍了 DeepSearch 和 DeepResearch 的概念、设计原则与实现细节。DeepSearch 通过持续的搜索、阅读和推理循环来寻找最优答案，而 DeepResearch 则是在 DeepSearch 基础上构建的用于生成长篇研究报告的框架。文章强调了长上下文 LLM、查询扩展、网页搜索和阅读能力的重要性，并分享了 Jina AI 在实际项目中所遇到的挑战，如报告质量、搜索结果可靠性等，以及如何通过 g.jina.ai 端点、查询扩展等方式解决这些问题。文章还深入探讨了系统提示词设计、知识空白问题处理、查询重写、网页内容爬取、内存管理、答案评估和预算控制等关键技术点，分享了 Jina AI 在项目实践中对向量模型、Reranker 模型以及 Agent 框架的取舍思考与选择，为读者提供了有价值的参考。

💡 主要观点

  1. DeepSearch 的核心在于搜索、阅读和推理的持续循环 通过多次迭代，系统能更深入地探索信息，评估结果，并优化搜索策略，从而找到更精准、更深入的答案。
  2. 长上下文 LLM 是 DeepSearch 和 DeepResearch 的基础 长上下文窗口使得系统能够记住之前的搜索尝试和网页内容，从而更好地规划后续操作，提高推理的准确性和连贯性。
  3. 查询扩展是决定搜索结果质量的关键因素之一 优秀的查询重写器能将自然语言转化为更适合搜索引擎处理的关键词形式，并扩展查询范围，覆盖更多潜在答案。
  4. 向量模型在去重任务中表现高效 利用 Embedding 模型计算语义文本相似度，可以轻松实现跨语言去重，避免非英语查询被误判过滤。
  5. 简化 Agent 框架，拥抱 LLM 原生能力 避免过度依赖框架，贴近 LLM 的原生能力可能更有效，减少开发负担。

💬 文章金句

  • DeepSearch 的核心理念是通过在搜索、阅读和推理三个环节中不断循环往复，直到找到最优答案。
  • DeepSearch 可以被视作一个配备了各类网络工具（比如搜索引擎和网页阅读器）的 LLM Agent。
  • 查询重写的重要性远超预期，甚至可以说是决定搜索结果质量的最关键因素之一。
  • 我们还发现，答案生成和评估放在不同的提示词中完成效果更好。
  • Agent 框架最终被证明是不必要的。在系统设计上，我们更倾向于贴近 LLM 的原生能力，避免引入不必要的抽象层。
