Azure Databricks 自动身份管理功能正式发布

2/18/2025, 7:00:00 PM

📌 一句话摘要

Azure Databricks 推出自动身份管理，通过与 Microsoft Entra ID 的无缝集成，简化用户管理和 AI/BI 仪表盘共享。

📝 详细摘要

Azure Databricks 正式发布自动身份管理功能，该功能集成了 Microsoft Entra ID，旨在简化用户和组的入职流程。工作区管理员可以直接通过 Databricks 界面添加 Entra ID 用户、组和服务主体，无需手动配置，也无需使用 SCIM 连接器。此功能简化了 AI/BI 仪表盘的共享，允许用户与组织内外任何 Entra ID 用户共享仪表盘，而无需额外的 Databricks 帐户。接收者可以使用其 Entra ID SSO 凭证登录，并自动获得访问权限。这有助于增强协作。该集成还通过集中通过 Entra ID 进行身份管理来增强安全性。

💡 主要观点

  1. 简化用户入职，提高管理效率 Databricks 工作区管理员可以直接从 Entra ID 导入用户和组，无需额外的配置步骤或自定义脚本，从而简化用户管理流程。
  2. AI/BI 仪表盘可以轻松地与任何 Entra ID 用户共享 用户可以与组织内外的 Entra ID 用户共享 AI/BI 仪表盘。接收者可以使用其 Entra ID SSO 凭证登录，而无需单独的 Databricks 帐户。
  3. 减少 IT 部门的负担，加速业务用户的数据访问 自动身份管理减少了 IT 部门处理用户权限请求的工作量，从而加快了业务用户的数据访问速度。

💬 文章金句

  • 自动身份管理现已在 Microsoft Entra ID 中提供受限公测版。该功能可实现与任何 Entra ID 用户、组或服务主体的即时、安全的身份配置，并支持仪表盘共享。
  • 仪表盘所有者现在可以轻松地与任何人（包括在 Entra ID 中注册的非 Databricks 用户）共享 AI/BI 仪表盘，以实现无缝协作。
  • 借助自动身份管理，AI/BI 仪表盘可以在整个组织内共享，甚至可以与 Databricks 工作区之外的团队（如非 Databricks 成员的业务用户）共享。
