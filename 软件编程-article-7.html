前端监控 SDK：从基础到实践 （1. 性能监控）

📌 一句话摘要

本文详细介绍了如何使用 PerformanceObserver API 和重写 fetch、xhr 等技术，构建前端监控 SDK 的性能监控模块，包括资源加载和关键性能指标的统计与上报。

📝 详细摘要

文章详细介绍了如何从零开始构建前端监控 SDK 的性能监控模块，并提供了实践指导。文章首先调研了两种统计页面性能数据的方法，最终选择了 PerformanceObserver API。接着，详细讲解了如何使用 PerformanceObserver API (如 PerformanceResourceTiming) 来捕获资源加载数据，并对数据进行过滤和加工，避免上报 SDK 自身的请求。然后，介绍了如何使用 PerformanceObserver API 来统计 FCP、LCP、FP 等关键性能指标。此外，文章还介绍了通过重写 fetch 和 xhr 来捕获 http 网络请求的方法，并给出了相应的代码实现， 带领大家从 0 到 1 搭建属于自己的监控平台。

💡 主要观点

  1. 通过 PerformanceObserver API，可以高效且标准化地采集 FP、FCP、LCP 等用户体验相关的核心性能指标。 PerformanceObserver 可以用来监测性能度量事件，获取 FP、FCP、LCP 等关键性能指标，为性能优化提供数据支撑。
  2. 通过重写 fetch 和 xhr，可以无侵入地监控 HTTP 请求，为分析接口性能提供数据基础。 重写 fetch 和 xhr 可以方便地获取网络请求的详细信息，包括请求的 URL、方法、状态码、耗时等，从而进行性能分析和问题排查。
  3. 前端监控 SDK 需要过滤自身请求，避免循环上报，保证数据的准确性。 由于数据上报是 SDK 的行为，可能会产生 ajax 请求，因此需要过滤掉这些请求，避免无限循环上报。
