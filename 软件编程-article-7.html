📌 一句话摘要

本文深入探讨大模型推理性能优化方法，如 Paged Attention、Radix Attention、Chunked Prefill 等，并分享 Deepseek-R1 高效部署实践。

📝 详细摘要

文章详细介绍了提升大模型推理性能的多种方法，并结合得物技术的实践经验进行评测分析。首先，文章阐述了高性能、易扩展的大模型推理框架的设计要点，强调 CPU 与 GPU 分离的重要性。接着，深入探讨了 Paged Attention 解决显存碎片问题、Radix Attention 缓存计算结果减少重复计算、Chunked Prefill 避免单个请求卡顿等优化策略。此外，文章还介绍了通过缩短输出长度、使用多卡推理以及预测解码等方式提升推理速度。最后，文章分享了使用 SGLang 推理引擎高效部署 Deepseek-R1 模型的具体步骤，展示了 SGLang 在简化部署流程中的作用。

💡 主要观点

  1. CPU 与 GPU 分离设计是实现高性能推理的关键 通过将 CPU 密集型任务与 GPU 计算分离，减少 GIL 锁竞争，显著提升 GPU 利用率和系统吞吐量。
  2. Paged Attention 有效解决显存碎片问题 借鉴操作系统内存管理机制，将 KV Cache 划分为固定大小的块，提升 GPU 利用率和推理吞吐量。
  3. Radix Attention 通过缓存共享前缀减少重复计算 利用基数树管理和重用不同请求之间共享的前缀，减少计算资源浪费，提升推理效率。
  4. Chunked Prefill 避免单个请求占用过多 GPU 资源 将 prompt 按照固定长度分块处理，减轻单个请求对 GPU 资源的占用，避免影响其他 decode 请求。
  5. 缩短输出长度，显著提升性能 通过限制最大输出长度、优化 prompt 或微调大模型等方式，减少 decode 阶段生成的 token 数量，从而显著提高整体响应时间。
  6. 预测解码利用小模型辅助大模型推理加速 通过小模型预测 token，大模型验证，或直接使用 n-gram 匹配 prompt，提升大模型推理速度。

💬 文章金句

  • Deepseek-r1 模型的爆火标志着本地部署大模型的需求日益增长。
  • 对于大模型推理来说，性能优化主要聚焦于两个关键指标：吞吐量与响应时间 (RT)。
  • Paged Attention 减少了内存浪费，提高了 GPU 的利用率，使模型的推理吞吐量比传统方法提升了数倍。
  • 在 vllm 的最新版本中，chunked prefill 已默认开启。
  • 可见推理中单卡变双卡可以显著提升大模型推理速度与 QPS。
