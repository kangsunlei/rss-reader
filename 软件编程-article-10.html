📌 一句话摘要

37signals 通过 Kamal 完成所有应用部署，实践其长期稳定和用户至上的承诺。

📝 详细摘要

文章讲述了 37signals 如何解决对云服务复杂性和高成本的顾虑，使用 Kamal 完成了所有应用程序的部署。Kamal 是作者 DHH 为了摆脱云服务限制而开发的轻量级部署工具，它简化了应用程序的部署流程，配置简便，并显著提升了部署效率，确保了应用程序的长期稳定性。37signals 强调了对用户的长期承诺，即使是 2003 年的旧版本 Basecamp 仍然在运行，并且可以使用 Kamal 进行部署。这种对用户信任的承诺是 37signals 的核心价值观，也是他们设计 Kamal 的重要考虑因素。文章还提到了 Docker 在实现应用程序长期运行方面的作用，Kamal 旨在成为 Docker 的简单封装和工作流。

💡 主要观点

  1. Kamal 简化了应用部署，降低了对云服务的依赖 Kamal 提供了一种简单、低成本的替代方案，避免了云服务的复杂性和高昂费用，使自建硬件部署应用成为可能。
  2. 37signals 承诺为用户提供长期稳定的服务 37signals 仍在运行 2003 年的旧版本 Basecamp，并使用 Kamal 进行部署，这体现了其对用户长期信任的技术承诺。
  3. Docker 实现了应用程序的封装和长期运行 Docker 允许封装整个系统及其依赖项，确保应用程序在任何环境中运行，并长期保持稳定。
  4. Kamal 实现了所有应用部署的统一性，包括遗留系统和新应用。 Kamal 提供了一种统一的部署模式，适用于 37signals 的所有 Web 应用，包括新应用和已维护二十年的遗留系统。

💬 文章金句

  • 我们摆脱云服务的计划因企业级的定价和 Kubernetes 的复杂性而陷入困境。
  • 最终确定了一种完全统一的部署模式，适用于我们在过去二十年中构建并仍在维护的每个 Web 应用。
  • 用户应该能够信任，当他们采用像 Basecamp 或 HEY 这样的系统时，不必担心因下一次管理层重组而被淘汰。
  • 要构建持久的东西，你必须使其简单。
  • Kamal 旨在成为围绕这种奇妙的简单性的一个简单封装和工作流。
