TypeScript 5.8 发布：改进类型推断，增强 JavaScript 生态系统互操作性

3/8/2025, 12:00:00 AM

📌 一句话摘要

TypeScript 5.8 发布，重点改进类型推断，增强 JavaScript 互操作性，并支持 Node.js 中直接运行。

📝 详细摘要

TypeScript 5.8 引入了更智能的条件返回类型推断，从而能够更准确地检测潜在的类型错误。新版本允许使用 `--erasableSyntaxOnly` 标志在 Node.js 中直接执行 TypeScript，前提是代码仅包含可擦除语法，但并非所有 TypeScript 代码都可以直接运行。使用 `--module nodenext` 标志，TypeScript 现在完全支持 JavaScript 模块的 `require()` 语法，从而增强了与现有 JavaScript 生态系统的兼容性。根据 TypeScript 专家 Matt Pocock 的说法，TypeScript 团队正在展望未来，可擦除语法将不再使用。这些改进旨在提高开发效率和互操作性。

💡 主要观点

  1. 改进的类型推断能够更准确地捕获条件返回类型中的错误。 TypeScript 5.8 增强了条件表达式的处理，检测到以前遗漏的类型错误，从而提高了代码质量。
  2. TypeScript 现在可以使用可擦除语法直接在 Node.js 中运行。 使用 `--erasableSyntaxOnly` 标志，可以无需编译即可运行 TypeScript 代码，从而简化了开发过程，但仅支持可擦除语法。
  3. `--module nodenext` 标志完全支持 JavaScript 模块的 `require()` 语法。 增强了在 TypeScript 中使用 `require()` 导入 JavaScript 模块时与现有 JavaScript 生态系统的兼容性。

💬 文章金句

  • 新版本延续了对类型推断的改进。
  • 此提案将减少构建步骤的需要，这可以使某些开发设置更加简单。用户可以直接运行他们编写的代码。
  • 使用 `--module nodenext` 标志，TypeScript 现在完全支持 JavaScript 模块的 `require()` 语法。
  • TypeScript 团队正在展望未来，这些语法将不再使用
