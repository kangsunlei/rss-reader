当前端工具链都在拥抱 Rust 时，这个知名前端库放弃 Rust，使用 TypeScript 重构，为啥？

2/17/2025, 8:17:54 AM

📌 一句话摘要

Prisma 因 Rust 引入的问题超过其性能优势，转而使用 TypeScript 重构，体现了技术选型需综合考虑各种因素。

📝 详细摘要

文章探讨了前端工具链拥抱 Rust 的趋势，并分析了 Prisma 放弃 Rust 转而使用 TypeScript 重构的原因。尽管 Rust 在性能上具有优势，但 Prisma 发现 Rust 引入了技术门槛、部署复杂性和兼容性问题。具体来说，Rust 要求贡献者同时掌握 Rust 和 TypeScript，限制了社区参与；需要为每个操作系统和 OpenSSL 版本生成独立的二进制文件，增加了部署难度；与现代 JavaScript 运行时、serverless 和边缘环境的兼容性较差，限制了 Prisma 的部署范围。Bun 的创始人 Jarred Sumner 认为，转向 TypeScript 将使 Prisma 更快，因为 NAPI 开销和事件循环调度开销对于数据库驱动程序来说非常重要。Prisma 的案例表明，技术选型需要在性能、易用性和生态系统之间做出权衡。

💡 主要观点

  1. Rust 性能优势明显，但引入了技术门槛和部署复杂性。 Rust 在处理大规模代码和并发场景下表现出色，但学习曲线陡峭，且需要为不同平台生成独立二进制文件。
  2. Prisma 放弃 Rust 的主要原因是兼容性和易用性。 Rust 二进制文件与现代 JavaScript 运行时、serverless 和边缘环境的兼容性较差，TypeScript 更易于集成和维护。
  3. Prisma 放弃 Rust 的部分原因是其与 serverless 和边缘计算环境的兼容性问题。 Rust 在 serverless 和边缘计算环境下的部署和运行存在挑战，而 TypeScript 更容易适应这些环境。

💬 文章金句

  • Rust 的性能和系统级特性使其成为构建核心查询引擎的理想选择。
  • 从 Rust 转向 TypeScript 将使 Prisma 更快。
  • NAPI 开销和事件循环调度开销对于 DB 驱动程序来说非常重要。
