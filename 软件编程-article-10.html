Node.js — Node v20.19.0 (LTS)

3/13/2025, 1:18:23 PM

📌 一句话摘要

Node.js v20.19.0 (LTS) 发布，主要更新包括默认启用 `require(esm)` 和模块语法检测，以及其他依赖更新和文档改进。

📝 详细摘要

Node.js v20.19.0 是一个特殊的次要版本更新，尽管 v20 处于维护模式，但由于 `require(esm)` 的重要性，该版本向后移植 (backport) 了此功能。此版本默认启用了 `require(esm)`，允许使用 `require()` 加载原生 ES 模块。同时，模块语法检测也默认启用，使得 Node.js 能够自动检测和运行包含 ES 模块语法的 JavaScript 文件。请注意，虽然模块语法检测提升了灵活性，但可能对 ES 模块的性能产生一定影响。此外，该版本还包括了加密模块的根证书更新、工作线程模块新增 `postMessageToThread` 方法，以及一系列依赖更新和文档改进。

💡 主要观点

  1. `require(esm)` 默认启用简化了 ES 模块的加载。 Node.js v20.x 版本不再需要通过命令行选项 (flag) 启用 `require(esm)`，使得 CommonJS 模块可以同步加载 ES 模块，提升了开发效率。
  2. 模块语法检测默认启用，提升了文件处理的灵活性。 Node.js 能够自动检测和运行包含 ES 模块语法的 JavaScript 文件，无需明确指定文件类型，简化了脚本的编写和执行。
  3. 该版本包含多个依赖更新和文档改进。 包括加密模块的根证书更新、simdutf 和 icu 等依赖的更新，以及文档的修订和改进，提升了 Node.js 的稳定性和安全性。

💬 文章金句

  • 通过 require() 加载原生 ES 模块的功能在 v20.x 版本中已通过命令行选项 --experimental-require-module 提供，并在 v22.x 和 v23.x 版本中默认提供。在此版本中，v20.x 版本不再需要通过该选项启用此功能。
  • 模块语法检测（`--experimental-detect-module` 选项）现在默认启用。
  • 用户可以检查 `process.features.require_module` 以查看当前 Node.js 实例中是否启用了 `require(esm)`。
