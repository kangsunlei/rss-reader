240 个标签页打乒乓？开发者用 Chrome 浏览器“整活”，网友：再玩内存要炸了！

2/25/2025, 10:54:02 AM

📌 一句话摘要

开发者利用 Chrome 浏览器标签页特性，通过 AppleScript、Web Worker 和 BroadcastChannel 等技术，实现了在多个标签页上运行乒乓球游戏的创新实践，用于展示前端技术的趣味性应用。

📝 详细摘要

本文介绍了一位开发者如何利用 Chrome 浏览器标签页，通过一系列技术手段，包括 AppleScript 自动化创建标签页网格、Web Worker 解决后台标签页刷新限制、BroadcastChannel 实现多标签页同步通信，将多个标签页作为一个整体进行控制和渲染，最终成功在 240 个标签页上运行乒乓球游戏《Pong》的创新实践。文章详细阐述了实现过程中的技术细节和遇到的挑战，以及如何通过优化减少资源占用，实现了相对流畅的游戏体验。这个项目展示了开发者充满想象力的创意和解决问题的能力，同时也引发了人们对浏览器资源占用和性能优化的思考。

💡 主要观点

  1. 利用 AppleScript 自动化创建和排列标签页网格，解决了手动操作的繁琐问题。 AppleScript 能够模拟用户在 macOS 系统上的操作，自动打开多个 Chrome 窗口并精确排列，为多标签页应用奠定了基础。
  2. Web Worker 突破了后台标签页的刷新限制，保证了游戏画面的实时更新。 Web Worker 允许在浏览器后台线程运行 JavaScript 代码，避免了主线程的阻塞，实现了在后台标签页中流畅更新 favicon。
  3. BroadcastChannel 实现了多标签页之间的同步通信，确保了游戏状态的一致性。 BroadcastChannel 是一种在同一域名下的不同标签页之间分发信息的方式，无需服务器即可实现标签页之间的实时通信和协调。
  4. 通过仅在 favicon 内容变化时才更新，避免了重复渲染，显著提升了游戏的性能。 避免在每一帧都重新生成 favicon，只在内容发生变化时才进行更新，有效降低了 CPU 占用率，提升了动画的流畅度。
