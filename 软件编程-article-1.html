Git 2.49 版本亮点

3/14/2025, 5:19:46 PM

📌 一句话摘要

Git 2.49 发布，重点改进包括更快的打包速度、部分克隆的历史数据块 (blob) 回填、zlib-ng 支持和 Rust 集成等。

📝 详细摘要

本文总结了 Git 2.49 版本中一些最有趣的新特性和改进。主要包括：1) 首次引入 Rust 代码，为 Git 的库化发展打下基础；2) 引入名称哈希 v2 (Name-hash v2) 提升打包速度和压缩率，特别是在包含大量同名但内容不同的文件时；3) 引入 `git backfill` 命令，优化部分克隆中历史数据块 (blob) 的回填，提升 `git blame` 等操作的性能；4) 支持使用 zlib-ng 提升压缩性能；5) `git gc` 命令支持 `--expire-to` 选项；6) 优化 `help.autocorrect` 配置；7) `git clone` 新增 `--revision` 选项；8) 移除对旧 remote 配置方式的支持，简化代码库，为未来功能铺路；9) 介绍了两位 Outreachy 实习生的贡献。

💡 主要观点

  1. Git 项目首次引入 Rust 代码。 引入了 libgit-sys 和 libgit 两个 Rust crates，为 Git 的库化发展打下基础。
  2. 名称哈希 v2 (Name-hash v2) 通过改进哈希算法显著提升 Git 的打包速度和压缩率。 通过更全面地考虑目录结构，新的哈希函数能更有效地识别相似文件，从而更好地进行增量 (delta) 压缩，减少存储空间。
  3. `git backfill` 命令优化了部分克隆的历史数据块 (blob) 回填。 通过批量请求和新的路径遍历 API (path-walk API)，`git backfill` 大大减少了回填缺失历史数据块 (blob) 所需的请求次数，提升了性能。
  4. Git 2.49 支持使用 zlib-ng 提升压缩性能。 zlib-ng 包含多种优化，并支持 SIMD 指令集，能显著提升 Git 的压缩和解压缩速度。
  5. Git 2.49 移除对旧 remote 配置方式的支持，为 Git 3.0 的发布做准备。 移除对旧 remote 配置方式的支持，简化了代码库，为未来功能铺平了道路。

💬 文章金句

  • Git 2.49 引入了一个新工具 `git backfill`，它可以通过少量的批处理，从 `--filter=blob:none` 克隆中填补任何缺失的历史数据块 (blob)。
  • 在 Git 2.49 中，您现在可以通过在使用 GNU Make 构建时传递 `ZLIB_NG`，或者在使用 Meson 构建时使用 `zlib_backend` 选项来使用 zlib-ng 构建 Git。
  • 此版本标志着 Git 项目的一个重要里程碑，首个 Rust 代码已检入。
