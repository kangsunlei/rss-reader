Cloud Trace 新功能：轻松解决延迟与错误

2/25/2025, 12:00:00 AM

📌 一句话摘要

介绍了 Google Cloud Trace 的新功能，通过改进的 UI 和分析后端，帮助开发者快速定位和解决应用程序的延迟和错误问题。

📝 详细摘要

文章介绍了 Google Cloud Trace 的更新，这是一个用于诊断应用程序延迟和错误的工具。新的 Trace 探索器 UI 建立在新的分析后端（BigQuery）之上，显著提升了性能，并计划推出 SQL 查询、灵活采样等新功能。Trace 探索器包含过滤器栏，用于选择 Google Cloud 项目的跟踪范围；一个多方面的跨度过滤器窗格，用于显示基于 OpenTelemetry 约定的常用过滤器；以及匹配跨度的可视化，包括交互式跨度持续时间热图、跨度速率折线图和跨度持续时间百分位图。文章通过一个零售网络商店的结账服务案例，展示了如何使用这些新功能来诊断和解决生产事件，尤其强调了 Trace 作用域 功能在跨项目问题排查中的作用。Cloud Trace 探索器现已正式上线。

💡 主要观点

  1. 新的 Trace 探索器 UI 助力用户快速定位性能瓶颈，诊断延迟问题。 热图、折线图和百分位图提供了不同的可视化视角，减少了排查问题所需的时间和精力，帮助用户快速识别异常跨度和性能瓶颈。
  2. Trace 作用域 功能允许用户跨多个 Google Cloud 项目进行跟踪。 通过设置跟踪范围，用户可以查看存储在其他相关项目中的跨度，从而更全面地了解应用程序的性能，解决跨项目问题。
  3. 基于 BigQuery 的新后端为 Cloud Trace 带来了更强大的分析能力。 用户可以期待 SQL 查询、灵活采样和区域存储等新功能，从而更有效地分析和管理跟踪数据。

💬 文章金句

  • 分布式追踪是可观测性栈的核心，助您轻松排查应用程序的延迟与错误。
  • 通过使用此 Trace 作用域 ，您将能够看到存储在与您的调查相关的其他 Google Cloud 项目中的跨度。
  • 这种新体验由 BigQuery 提供支持，使用与日志分析相同的平台。
