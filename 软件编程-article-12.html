Grafana Mimir 下一代架构：深入解析可靠性提升的设计

2/20/2025, 11:58:06 AM

📌 一句话摘要

Grafana Mimir 通过集成 Kafka 和 WarpStream，重塑架构，提升可靠性，以支持未来增长和新应用。

📝 详细摘要

本文介绍了 Grafana Mimir 的下一代架构，旨在提高其可靠性。当前架构的 ingester 节点是有状态的，导致读取和写入操作相互干扰。为了解决这个问题，新架构分离了读取和写入路径，使用 Apache Kafka 作为其基础。对于 Grafana Cloud Metrics，引入了 WarpStream，一个基于对象存储的 Kafka 兼容数据流平台，以更好地为具有严格 SLA 要求的客户提供服务，并降低跨可用区部署成本。负载测试证实 WarpStream 满足扩展需求。新架构正逐步推广到 Grafana Cloud Metrics，并最终将推广到 Mimir OSS，有望提高对查询流量和应对数据摄入高峰的能力。

💡 主要观点

  1. 新架构增强了 Mimir 处理查询流量和数据摄入高峰的能力。 通过引入 Kafka 和 WarpStream，Mimir 的弹性得到增强，使其能够更好地处理高负载场景，并更好地应对突发流量高峰。
  2. Mimir 的新架构旨在分离读取和写入路径，从而提高系统可靠性。 通过分离读写操作，避免了繁重的查询负载对实时写入的影响，并降低了 ingester 节点的管理复杂性。
  3. WarpStream 降低了 Grafana Cloud Metrics 跨可用区部署的成本。 WarpStream 构建在对象存储之上，消除了跨可用区网络传输成本，从而实现了更经济的多可用区部署。

💬 文章金句

  • 在现有架构中，ingester 节点是有状态的，并将内存中的 TSDB 与存储在持久卷上的本地预写式日志 (WAL) 相结合。
  • 我们选择了 Apache Kafka，一个由 Apache 软件基金会开发的开源分布式事件流平台，作为 Mimir 重新设计的基础，因为它是具有蓬勃发展的生态系统的开放且不断增长的标准。
  • WarpStream 引起了我们的注意，因为它是一个基于对象存储的 Kafka 兼容数据流平台。
