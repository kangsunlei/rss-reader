📝 详细摘要

本文深入解析 MyBatis 四大核心组件：Executor、StatementHandler、ParameterHandler 和 ResultSetHandler 的功能、实现原理及相互关系，并结合源码分析和实例演示，帮助开发者理解 MyBatis 底层原理，从而进行更高效的开发和优化。

💡 主要观点

  1. Executor 负责 SQL 执行和事务管理，是 MyBatis 的核心。 Executor 通过管理 StatementHandler 来执行 SQL，并负责事务的提交和回滚，以及 MyBatis 的缓存处理。
  2. StatementHandler 负责 Statement 的创建、参数绑定和结果处理。 StatementHandler 根据 SQL 类型选择不同的实现类，如 PreparedStatementHandler 和 SimpleStatementHandler，并调用 ParameterHandler 和 ResultSetHandler 完成参数绑定和结果转换。
  3. ParameterHandler 负责将 Java 对象转换为 SQL 参数。 ParameterHandler 通过 TypeHandler 将 Java 类型转换为 JDBC 类型，并将参数设置到 PreparedStatement 中。
  4. ResultSetHandler 负责将结果集转换为 Java 对象。 ResultSetHandler 遍历结果集，并使用构造函数和映射关系将数据填充到 Java 对象中。

💬 文章金句

  • ResultSetHandler 主要用于将查询结果转换成 Java 对象；而 ResultHandler 主要用于对查询结果以某种形式展现。它们的使用场景是不同的
  • Executor 执行器的核心功能，就是新建出一个 statementHandler ， 然后利用该 handler 创建一个 statement 的实例。最后调用该实例的对应方法完成 sql 的执行。
  • ParameterHandler 其实就是两个功能，一个是提供入参对象，另一个就是把入参给 sql 注入进去
