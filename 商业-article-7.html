从放弃的 AI 浏览器到通用 Agent：完整复盘 Manus 的诞生过程

3/12/2025, 12:50:34 PM

📌 一句话摘要

蝴蝶效应团队通过 Manus 率先实现了通用 Agent 产品，并对模型对齐提出了深刻见解，该产品通过异步并行执行任务，体现了 Less Structure，More Intelligence 的产品理念。

📝 详细摘要

文章回顾了蝴蝶效应团队从 AI 浏览器项目到通用 Agent 产品 Manus 的转型历程。团队在 AI 浏览器开发中发现 AI 会打断用户体验，转而为 AI 打造专属云端浏览器。Manus 的设计理念是“Less Structure，More Intelligence”，通过异步并行执行任务，减少人工干预，让 AI 自主探索和学习。文章深入探讨了 Agent 领域的核心问题——模型对齐，强调需要针对 Agent 场景进行专门的训练和优化。Manus 在 GAIA Benchmark 上表现优异，并能覆盖现有 Agent 产品的多种应用场景，展示了其在通用 Agent 领域的潜力。文章总结了蝴蝶效应团队在“套壳”模式下的创新经验，以及对模型驱动和快速迭代的深刻理解，并展望了 Manus 开源后的潜在影响。

💡 主要观点

  1. Manus 核心理念是 Less Structure，More Intelligence。 减少人工干预，让 AI 通过自主探索和学习来完成任务，最大化发挥 AI 的潜力，实现超预期的效果。
  2. AI 浏览器开发的教训是 AI 应有自己的云端浏览器。 团队发现 AI 在用户浏览器中会打断用户体验，因此转向为 AI 打造专属的云端环境，解决资源抢占和安全问题。
  3. Agent 的关键是对齐，而非基础模型能力。 针对 Agent 场景进行专门的训练和优化，解决现有模型在规划和多轮交互中的不足，提升 Agent 的实用性。
  4. Manus 模拟的是能干事的人，而非特定角色。 Manus 旨在模拟实习生的工作方式，通过调用各种工具完成任务，而不是模拟研发、产品经理等特定角色。
  5. 快速创新是在全新领域成功的关键。 在 Agent 领域，快速探索、试错，并快速找到正确的路径至关重要，蝴蝶效应团队通过灵活的管理和决策机制实现快速创新。

💬 文章金句

  • Agent 可能是『对齐』的问题，而不是基础模型能力的问题
  • AI 浏览器不是在浏览器里加 AI，而是做给 AI 用的浏览器
  • 今天的模型能力是能够完成一些复杂、多步才能搞定的任务。只是没有这样的产品，所以大家感受不到。
  • 在一个全新的领域里，一切未定，一切未知，最重要的就是创新速度，拼的是在各种各样的方向上探索、试错，快速找到正确的路径。
  • 在做一款产品的时候，也会频繁关注外界的情况。当时有几件事，一是做浏览器的时候，做过端侧模型，后来发现浏览器需要的场景非常非常宽，有不同的 feature，做的过程发现了基座模型变强的速度在加快，强到它跟 agent 之间的差距可能是一个 alignment 问题。
