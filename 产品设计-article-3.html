解决 mTLS 会话恢复漏洞

2/7/2025, 8:13:14 PM

📌 一句话摘要

Cloudflare 修复了一个 mTLS 会话恢复漏洞,该漏洞允许未经授权的客户端绕过身份验证。

📝 详细摘要

Cloudflare 解决了影响使用 mTLS 客户的 mTLS 会话恢复漏洞。攻击者可以利用此漏洞,使用来自一个 Cloudflare 区域的有效 mTLS 证书,在未经验证的情况下,恢复与另一个 Cloudflare 区域的 TLS 会话。该漏洞已通过禁用 mTLS 客户的 TLS 会话恢复来缓解。该漏洞利用涉及更改 TLS 服务器名称指示 (SNI) 和 HTTP Host 标头。Cloudflare 正在探索恢复 mTLS 客户的 TLS 会话恢复性能改进的方法,并建议客户使用转换规则、日志记录和防火墙功能进一步加强其 mTLS 配置。

💡 主要观点

  1. Cloudflare 已通过禁用 mTLS 的 TLS 会话恢复来缓解该漏洞 作为补救措施,Cloudflare 已禁用 mTLS 客户的 TLS 会话恢复,以防止会话缓存客户端证书及其验证状态。
  2. mTLS 会话恢复漏洞允许未经授权的客户端绕过身份验证 攻击者可以利用来自一个区域的有效证书来恢复与另一个区域的会话,从而绕过第二个区域所需的 mTLS 身份验证。
  3. 客户可以通过其他方法进一步加强其 mTLS 配置 Cloudflare 建议使用转换规则、日志记录和防火墙功能来加强 mTLS 配置,以检测未来的问题。

💬 文章金句

  • 然而,在 Cloudflare 的 mTLS 实现中,会话恢复引入了一种意想不到的行为。
  • 我们已为所有启用 mTLS 的客户禁用了 TLS 会话恢复。
  • 安全是我们 Cloudflare 一切工作的核心,对于此问题可能造成的任何困扰,我们深感遗憾。
