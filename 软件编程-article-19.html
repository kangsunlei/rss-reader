从 ClickHouse 到 Apache Doris：在网易云音乐日增万亿日志数据场景下的落地

2/28/2025, 12:37:36 PM

📌 一句话摘要

网易云音乐使用 Apache Doris 替换 ClickHouse 构建日志平台，有效支撑日增万亿日志数据，并实现性能提升和成本降低。

📝 详细摘要

文章介绍了网易云音乐从 ClickHouse 迁移到 Apache Doris 构建日志平台的实践。早期基于 ClickHouse 的日志平台面临运维成本高、并发查询能力不足、写入性能不稳定等问题。为解决这些问题，网易云音乐引入 Apache Doris，Doris 具备高并发、低延迟等优势，并针对存储设计、写入和查询进行了优化改造，例如采用随机分桶、ZSTD 压缩、倒排索引等策略。同时，针对 Flink Connector 和 FE 元数据的调优，实现了查询响应速度提升、并发能力增强、写入稳定性提高和运维成本降低。Doris 支持冷热存储，能够将冷数据存储到 S3 或 HDFS 等低成本存储介质，进一步降低存储成本。最终，新日志平台稳定运行，有效支撑了日增万亿级别日志数据的场景。

💡 主要观点

  1. Apache Doris 替换 ClickHouse 可显著降低运维成本 Doris 易于运维、支持横向扩展，结合自恢复能力和自动均衡能力可有效降低人工干预，简化运维流程。
  2. 针对 Doris Flink Connector 的优化可提升写入性能 通过直接写入压缩流、开启单 tablet 导入、随机选择 BE 节点写入等方式，降低内存占用，提升写入速度和均衡性。
  3. 利用倒排索引和全文检索函数可加速日志查询 Doris 提供的倒排索引和全文检索 MATCH 查询性能优于 LIKE 模糊匹配，并具备自动大小写和单复数归一化能力。
  4. 调整参数可使磁盘之间更加均衡 通过关闭 trash、调整磁盘选择策略等参数设置，解决磁盘 I/O 不均衡问题，提升系统整体性能。

💬 文章金句

  • 在新架构中，使用 Apache Doris 替代 ClickHouse 作为日志存储和分析引擎。
  • 由于 ClickHouse 和 Doris 均采用关系数据库模型及 SQL，因此架构变化很小、迁移也比较简单。
  • Doris 的数据湖扩展能力也是网易云音乐非常看重的能力，这为规划中的 One-SQL 数据架构奠定了坚实基础。
  • 支持将冷数据存储到 S3 或 HDFS 等低成本存储介质，存储成本进一步降低。
