Node.js - EOL 版本 CVE 更新

3/7/2025, 4:00:00 PM

📌 一句话摘要

Node.js 更新 CVE 策略，将 EOL 版本纳入范围，旨在解决 MITRE 拒绝为 EOL 版本发布新 CVE 的问题，并警示用户 EOL 版本的安全风险。

📝 详细摘要

Node.js 团队更新了处理停止维护 (EOL) 版本 CVE 的方法。最初，针对 EOL 版本发布了特定的 CVE（CVE-2025-23087、CVE-2025-23088、CVE-2025-23089），但这些 CVE 被 MITRE 拒绝。因此，Node.js 决定更新现有的漏洞 CVE，以涵盖 EOL 版本，以反映其安全风险。这意味着所有新的 CVE 都将包括 EOL 版本，除非有特定信息表明情况并非如此。由于资源限制，Node.js 不会主动评估 EOL 版本的安全报告，但可能会根据外部提供的信息更新适用性。对于需要继续使用 EOL Node.js 版本的组织，OpenJS 生态系统可持续性计划提供商业支持选项。

💡 主要观点

  1. Node.js 扩展了所有新 CVE 的范围，以包括 EOL 版本。 为了反映与停止维护 (EOL) 版本相关的持续安全风险，Node.js 决定将 EOL 版本包括在所有新的 CVE 中，除非有特定信息表明 CVE 不适用于 EOL 版本。
  2. MITRE 拒绝了最初为 EOL 版本发布的特定 CVE。 MITRE 将这些 CVE 标记为“分配时未提供支持”，并将其标记为“有争议”，因为它们没有明确指出具体的漏洞。
  3. 由于资源限制，Node.js 不会主动评估 EOL 版本的安全性，但会根据外部提供的信息更新 CVE 适用性。 由于资源限制，Node.js 不会主动评估 EOL 版本的安全报告，但会根据外部来源提供的信息更新 CVE 适用性。

💬 文章金句

  • 为什么 Node.js 项目为所有 EOL 版本发布 CVE？因为我们没有资源评估每一个过去的版本，以了解哪些版本存在漏洞。
  • Node.js 由志愿者运营。我们有足够的资金来维护当前的版本，但不能超出这个范围。换句话说，所有过去的 Node.js 版本都存在漏洞或很快就会存在漏洞。
  • 这个 CVE 突出了贵组织面临的风险。
