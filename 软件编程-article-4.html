Vercel Fluid：颠覆无服务器的新型计算模型？

3/9/2025, 12:00:00 AM

📌 一句话摘要

Vercel Fluid：通过单工作进程多请求处理优化资源利用率的弹性计算模型。

📝 详细摘要

Vercel Fluid 是一种弹性计算模型，允许单个工作进程处理多个请求，类似于传统服务器，同时保持无服务器的弹性。它旨在提升传统无服务器架构在处理长时间运行任务和 AI 推理时的效率。Fluid 优先利用现有资源，再扩展新资源，有望缩短冷启动时间，提升资源利用率。该模型利用 `waitUntil` API 处理 HTTP 响应后的任务，并采用按需付费的定价模式。但是，用户反馈表明对定价模型和实际成本效益存在一些不确定性。

💡 主要观点

  1. Vercel Fluid 通过单工作进程多请求处理优化资源利用率，区别于传统无服务器架构。 与传统为每个请求分配虚拟机的无服务器架构不同，Fluid 允许多个并发请求共享一个虚拟机，从而提高资源利用率，尤其是在 I/O 密集型场景中。
  2. `waitUntil` API 支持 HTTP 响应后任务处理，从而增强用户体验。 Fluid Compute 使用 `waitUntil` API 来处理发送 HTTP 响应后的任务，从而减少延迟并提高感知性能，而不会影响响应时间。
  3. 定价模型和实际成本效益是用户关注的关键问题。 尽管 Vercel Fluid 声称可以降低计算成本，但用户对其定价模型和实际成本效益仍持谨慎态度，需要进一步验证和提高透明度。
  4. Vercel Fluid 的出现对 AWS Lambda 等现有无服务器平台构成竞争压力。 Fluid Compute 在成本效率方面提供了竞争优势，从而引发了其他无服务器提供商的响应和潜在创新。

💬 文章金句

  • Vercel 融合了服务器在成本效益和运行时能力上的优势，以及无服务器的开发者体验 (DevX) 与安全性。
  • 最大的区别在于微虚拟机的利用方式。Lambda 保留整个虚拟机来处理端到端的请求。Fluid 可以使用一个虚拟机来处理多个并发请求。由于大多数工作负载通常在等待 IO 时处于空闲状态，因此最终效率会更高。
  • 无论 AWS Lambda 是否已经弄清楚函数内并行，或者已经弄清楚但只是选择不“传递节省的成本”，Fluid Compute 在成本效率方面都提供了竞争优势，我很难想象 Lambda 不会跟上。
  • 对于像我这样在 Vercel 上运营前端的适度 SaaS 业务的人来说，定价模式的改变只是这些天疲劳的另一个来源。
