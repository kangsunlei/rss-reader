📌 一句话摘要

JetBrains Rider 2025.1 EAP 5 发布，引入 Roslyn 语法树可视化工具、Unity Profiler 集成、增强的 C++ 调试 (Natvis 文件处理) 等新功能。

📝 详细摘要

JetBrains Rider 2025.1 EAP 5 版本发布，引入 Roslyn 语法树可视化工具，方便开发者检查 C# 代码结构，以及 Unity Profiler 集成，可以直接在编辑器中查看 CPU 性能数据。此外，该版本还为 Roslyn 分析器引入了范围性快速修复，可以批量修复文件、项目或解决方案中的问题；支持 `dotnet watch` 作为运行 / 调试配置，简化了 ASP.NET Core 和 .NET 开发的自动构建和调试流程；改进了运行配置的输入 / 输出重定向功能，方便调试和测试；增强了 Windows 上 C++ 调试的 Natvis 文件处理能力。这些新功能旨在提升 .NET 和 Unity 开发者的效率。

💡 主要观点

  1. Unity Profiler 集成将 CPU 性能洞察直接引入 Rider 编辑器。 Rider 可以获取 CPU 使用快照，并在代码中内联显示，方便开发者在编写和调试 Unity 脚本时定位性能热点。
  2. Roslyn 语法树可视化工具便于理解 C# 代码的解析和结构。 该工具允许开发者在 IDE 中直接检查语法树，通过动态更新、智能导航和搜索过滤功能，更轻松地理解 C# 代码的语法结构和组成。
  3. 支持 `dotnet watch` 作为运行/调试配置，简化了开发流程。 此增强功能特别有利于 ASP.NET Core 和 .NET 开发者，因为它可以在修改 `.cs` 、 `.html` 或 `.css` 文件时持续构建应用程序。

💬 文章金句

  • Rider 现在集成了 Unity Profiler，将 CPU 性能洞察直接引入您的编辑器。
  • Rider 的全新 Roslyn 语法树可视化工具让您可以直接在 IDE 内部检查语法树，从而更轻松地理解代码的解析方式和结构。
  • Rider 2025.1 EAP 5 为 `dotnet watch` 引入了专用运行和调试配置，从而简化了需要随着代码更改自动重建和重启的项目的开发工作流程。
