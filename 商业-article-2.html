“3000 万行 C 代码暂时不会消失，新内核驱动可用 Rust 写写看”，Linux 二把手表态支持 Rust！

2/20/2025, 10:19:18 AM

📌 一句话摘要

Linux 内核二把手 Greg Kroah-Hartman 表态支持使用 Rust 编写“新”的内核代码和驱动程序，以避免 C 语言中的内存安全漏洞。

📝 详细摘要

文章主要讲述了 Linux 社区关于是否在内核开发中引入 Rust 语言的争议和进展。尽管一些维护者对 Rust 持批评态度，但 Linux 内核维护者 Greg Kroah-Hartman 公开表示支持使用 Rust 编写新的内核代码和驱动程序。他认为 C 语言存在许多内存安全漏洞，而 Rust 可以有效避免这些问题。Linus 倾向于强制合并 Rust 代码，并制定了内核中 Rust 代码的维护策略。尽管社区对 Rust 的接受程度不一，但 Linux 似乎正在为应用 Rust 做好准备。

💡 主要观点

  1. Linus Torvalds 倾向于推进 Rust 代码的合并 尽管一些维护者反对，Linus Torvalds 私下表示将推进 Rust 代码的合并，这意味着 Rust 在 Linux 内核中将扮演更重要的角色。
  2. Linux 内核维护者 Greg KH 支持使用 Rust 编写新代码 Greg KH 认为 Rust 可以避免 C 语言中常见的内存安全漏洞，从而提高内核的稳定性和安全性。
  3. 社区对 Rust 的接受程度不一 一些开发者认为 Rust 的优势可以通过其他方式实现，并且 Rust 代码的可读性也存在问题，但也有人认为引入 Rust 可以为 Linux 注入新的活力。

💬 文章金句

  • 我们遇到的大多数 bug（按数量而非质量/严重性来算）都是由于 C 语言中的愚蠢的小极端情况造成的，而这些在 Rust 中完全消失了
  • 我们有 3000 万行 C 代码，这些代码在未来几年内不会消失。这是一项值得的努力，不会停止，也不应该停止。
  • Rust 并不是能解决所有问题的“银弹”，但它肯定能在许多地方提供巨大帮助。那么，为什么我们不想要它呢？
  • 加入另一种语言真的不应该是问题，我们过去处理过更糟糕的事情，我们不应该放弃现在确保我们的项目能够在未来 20 多年继续成功的机会。
