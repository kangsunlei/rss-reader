📝 详细摘要

文章揭秘了 DeepSeek V3/R1 大语言模型的推理系统，重点介绍了其在吞吐量和延迟优化方面的设计原则和实现方法。文章详细阐述了跨节点专家并行（EP）策略，以及如何通过扩展 batch 大小、隐藏通信延迟和实现负载平衡来应对 EP 带来的挑战。此外，文章还公开了 DeepSeek 在线服务的统计数据，包括每个 H800 节点的 token 处理能力、成本利润率以及推理服务节点的使用情况。DeepSeek 针对预填充和解码阶段，以及专家并行，分别设计了负载平衡器。最后，文章分析了 DeepSeek 实际收入与理论收入之间的差异，并解释了定价策略和非高峰时段折扣等因素的影响。通过这些信息的披露，DeepSeek 旨在为社区带来价值，并推动 AGI 的发展。

💡 主要观点

  1. 专家并行（EP）提升吞吐量和降低延迟 通过将专家分布到多个 GPU 上，并扩展 batch 大小，EP 策略能够提高 GPU 的计算效率，并降低内存访问需求，从而提升整体性能。
  2. 计算 - 通信重叠技术有效缓解大规模并行化带来的通信开销 DeepSeek 采用 dual-batch 重叠策略和流水线技术，将通信成本隐藏在计算过程中，从而提高整体吞吐量。
  3. 实现最优负载平衡是最大限度提高资源利用率的关键 DeepSeek 针对预填充和解码阶段，以及专家并行，分别设计了负载平衡器，确保所有 GPU 上的计算和通信负载均衡，避免出现性能瓶颈。
  4. DeepSeek 通过 FP8 和 BF16 混合精度计算优化服务性能 矩阵乘法和分发传输采用与训练一致的 FP8 格式，而核心 MLA 计算和组合传输使用 BF16 格式，确保最佳服务性能。
