VisualStudio.Extensibility：Tagger 支持与 Settings API 更新

2/18/2025, 5:12:06 PM

📌 一句话摘要

VisualStudio.Extensibility SDK 17.13 发布，重点在于增强编辑器可扩展性的 Tagger 支持，以及用于监控设置变更的扩展 Settings API。

📝 详细摘要

本文介绍了 VisualStudio.Extensibility SDK 17.13，重点介绍两个主要功能。首先是通过 Tagger 支持增强的编辑器可扩展性，允许开发人员向 Visual Studio 本身不支持的语言添加自定义文本装饰器，例如语法突出显示和 CodeLens。本文提供了用于定义 CodeLens 标签的示例代码，并链接到完整的代码示例和文档。其次是扩展的 Settings API，允许扩展通过设置观察器来监视设置值的变更。文章还提到这些 API 目前是实验性的，意味着随着 API 根据反馈不断发展，可能会出现非兼容性变更。鼓励开发者通过 GitHub Issue Tracker 和开发者社区提供反馈，以帮助改进 VisualStudio.Extensibility SDK。

💡 主要观点

  1. Tagger 支持增强了编辑器可扩展性 允许开发人员向 Visual Studio 本身不支持的语言添加自定义文本装饰器（如 CodeLens），从而扩展 Visual Studio 的功能。
  2. 扩展的 Settings API 允许观察设置值的变更 扩展可以通过设置观察器来监视设置值的变更，从而对设置变更做出反应并调整其行为。
  3. VisualStudio.Extensibility SDK 17.13 通过 Tagger 支持和 Settings API 扩展增强了 Visual Studio 扩展的性能和可靠性。 该 SDK 旨在通过在 Visual Studio IDE 进程之外运行扩展来提高性能、可靠性和安装速度。
  4. Tagger 支持和扩展的 Settings API 以实验性功能的形式提供 允许开发人员在实际应用程序中提供反馈并参与 API 的设计迭代。

💬 文章金句

  • VisualStudio.Extensibility 帮助您构建在 Visual Studio IDE 进程之外运行的扩展，从而提高性能、可靠性和安装速度，而无需重启 Visual Studio。
  • Tagger 是一种用隐藏信息标记编辑器文本的机制，使编辑器能够应用各种文本装饰。
  • 我们在 17.13 中引入的另一个新增功能是对 Settings API 的增强，它允许扩展开发者收到其设置值变更的通知。
