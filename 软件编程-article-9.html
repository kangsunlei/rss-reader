Databricks Lakehouse 联邦正式发布 Google BigQuery 版本，Teradata 和 Oracle 版本进入公开预览阶段

3/6/2025, 1:24:25 PM

📌 一句话摘要

Databricks Lakehouse 联邦迎来扩展，Google BigQuery 版本正式发布，Teradata 和 Oracle 版本进入公开预览。无需数据迁移即可实现统一数据访问和统一数据管理。

📝 详细摘要

本文宣布 Databricks 中 Lakehouse 联邦正式发布 Google BigQuery 版本，公开预览 Oracle 和 Teradata 版本。Lakehouse 联邦允许用户跨各种数据库和数据仓库查询和分析数据，而无需数据重复或传输，并通过 Unity Catalog 实现统一数据管理。主要优势包括统一的数据视图、无缝查询和一致的安全策略。与 Google BigQuery 的集成支持直接从 Databricks 在不同云环境（GCP、Azure、AWS）上执行 BigQuery 数据的 workload，而 Teradata 和 Oracle 的新连接器扩展了 Lakehouse 联邦的范围。它简化了数据操作并增强了跨多云策略的分析。

💡 主要观点

  1. Lakehouse 联邦统一了跨平台的数据访问。 它允许在不同的系统中查询数据而无需 ETL，从而创建统一视图并简化数据分析。
  2. BigQuery 连接器 GA 版本为多云环境下的高效分析提供支持。 用户可以直接在 Databricks 中跨 GCP、Azure 和 AWS 运行 BigQuery 数据的 workload，并利用 Unity Catalog 进行治理。
  3. 新连接器扩展了 Lakehouse 联邦对 Teradata 和 Oracle 的支持。 Teradata 和 Oracle 连接器的公开预览版本进一步增强了数据集成能力，实现数据统一管理。

💬 文章金句

  • 现在，您可以通过 Unity Catalog 连接、发现、管理和查询来自这些来源的数据，而无需迁移或 ETL。
  • 获得整个数据资产的统一视图：在单个平台中自动分类和发现结构化和非结构化数据，助力您的组织安全地访问和探索所有可用数据，打破数据孤岛。
  • Lakehouse 联邦 for BigQuery 已经无缝连接了我们的 Google Cloud 和 Azure 环境。无需数据移动或重复，我们现在可以利用 Azure 上的 Databricks 直接在 BigQuery 数据上运行 Python 和 SQL 工作负载，同时受益于 Unity Catalog 的细粒度治理。
