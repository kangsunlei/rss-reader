TypeScript 官方宣布弃用 Enum？Enum 何罪之有？

3/8/2025, 12:02:47 PM

📌 一句话摘要

本文探讨了 TypeScript Enum 的争议和替代方案，以及 `erasableSyntaxOnly` 配置与 Node.js 兼容性的关系。

📝 详细摘要

文章深入探讨了 TypeScript 中 `enum` 的使用争议和替代方案。首先，文章介绍了 TypeScript 5.8 版本中新增的 `--erasableSyntaxOnly` 配置项，该配置项只允许使用可擦除语法，而 `enum` 属于不可擦除语法，因此引发了关于官方是否不再推荐使用 `enum` 的讨论。接着，文章分析了 `enum` 的三大罪行，包括默认值和隐式类型转换带来的类型安全问题、不支持枚举值字面量以及增加运行时开销。最后，文章提出了四种替代方案，包括 `const enum` (解决了运行时开销问题，但不支持字面量)、模板字面量类型 (支持字面量，但仍有运行时开销)、联合类型 (限制取值，但缺乏统一维护) 以及类型字面量 + `as const` (兼具了字面量和类型安全的优点)，并分析了它们的优缺点。文章总结认为，官方推出 `--erasableSyntaxOnly` 配置是为了兼容 Node.js 的新特性，而不是要放弃 `enum`。

💡 主要观点

  1. `erasableSyntaxOnly` 配置是为了更好地与 Node.js 运行时环境集成，并非要弃用 `enum`。 Node.js 23.6.0 默认支持直接执行可擦除语法的 TS 文件，该配置项用于禁用不可擦除语法，以减少运行时开销。
  2. Enum 存在类型安全问题、不支持字面量及增加运行时开销等问题。 Enum 默认值会带来类型安全隐患，不支持枚举值字面量导致使用不便，编译后生成双向映射数据增加运行时开销。
  3. 字面量类型 + `as const` 是 `enum` 的一种常用且灵活的替代方案。 该方案既可以传入枚举值，又可以传入枚举类型，兼具字面量和类型安全的优点。

💬 文章金句

  • TS 可擦除语法是指 type、interface、n:number 等可以直接去掉的、仅在编译时存在、不会生成额外运行时代码的语法
  • TS 不可擦除语法是指 enum、constructor(public x: number) {} 等不可直接去除且会生成额外运行时代码的语法
  • 用字面量类型 + as const 是比较常用的一种方案。
