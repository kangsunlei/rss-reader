探索内在 Web 设计 (IWD)，旨在超越传统响应式 Web 设计 (RWD)，利用浏览器原生能力实现更灵活的声明式布局。

📝 详细摘要

本文回顾了响应式 Web 设计 (RWD) 的演变，并介绍了内在 Web 设计 (IWD) 的概念，它利用现代布局系统（如 Flexbox 和 CSS Grid）来允许浏览器更多地参与布局决策，以实现更灵活的自适应布局。文章指出，尽管 IWD 尚未像 RWD 那样成为主流，但其核心思想——信任浏览器并采用声明式设计——已逐渐渗透到 Web 开发实践中。作者还分享了语义化 HTML、流式排版和渐进增强等实用建议，并提倡在使用 rem 和 em 单位时采用比例思维，以更好地拥抱 IWD 理念。IWD 代表了 Web 设计理念的转变，鼓励开发人员拥抱浏览器的能力，以实现更具适应性和可维护性的 Web 应用程序。

💡 主要观点

  1. IWD 的核心是信任浏览器并采用声明式设计 将部分布局控制权交给浏览器，开发人员只需要声明“做什么”而不是“如何做”，从而允许浏览器根据用户的环境进行优化。
  2. 语义化 HTML 和渐进增强是 IWD 的重要实践 确保内容在各种环境下的可访问性，并利用现代浏览器功能来提供更好的体验，同时保持向后兼容性。
  3. 采用比率思维进行布局设计 避免将相对单位转换为固定像素值，保持布局的灵活性，并更好地适应不同的用户设备和设置。

💬 文章金句

  • Instead of telling it *how* to do things, we rather tell it *what* to do and leave it to figure out the "how" part by itself.
  • Focus on creating the right *inputs* rather than trying to control every possible *output*.
  • Be the browser's mentor, not its micromanager.
  • Trust it because it knows much more about how safe it is for users to surf the web.
