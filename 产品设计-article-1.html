B 端中后台 UI 设计规范定义&实践经验总结

3/9/2025, 4:35:14 PM

📌 一句话摘要

本文总结了 B 端中后台 UI 设计规范的定义和实践经验，重点介绍了原子设计理论和 Design Token 的应用，旨在提升产研团队的设计质量和协作效率。

📝 详细摘要

文章详细阐述了 B 端中后台 UI 设计规范的定义与实践，首先分析了搭建设计规范对产品、设计、开发和测试的意义。接着引入原子设计理论，将界面设计拆解为原子、分子、组织、模板和页面五个层面。然后深入探讨了后台设计系统的搭建，包括色彩、文字、阴影、圆角和线条等原子元素，以及分子和组织的组合方式。最后重点介绍了 Design Token 在解决开发还原准确度（减少开发走查轮次）、主题切换（降低主题切换成本）和设计变更维护等问题上的应用，并给出了 Design Token 的应用流程和命名规则。通过设计规范和 Design Token 的应用，可以有效提高产研团队的设计质量和协作效率。

💡 主要观点

  1. 原子设计理论能有效提升设计和开发的协作效率 通过将设计拆分成原子、分子、组织等基本元素，再根据业务需求重新组装，形成最终产品，实现组件化设计，区别于整体设计制造。
  2. Design Token 可以解决产品设计和开发过程中的一致性问题 Design Token 通过规定样式参数，并采用一套统一的命名规则，有效提高产研团队设计质量和协作效率，并实现主题一键切换。
  3. 提炼 Token 元素和定义命名规则是应用 Design Token 的关键步骤 通过提炼 Token 元素，定义由大到小的命名规则，并整理 Design Token 资产表，可以更好地统一规范，高效应用 Token。

💬 文章金句

  • 为了更好的统一规范，应用 Token，建议成熟的互联网公司设计团队搭建自己的低代码平台。
  • 标准化的设计规范，是测试人员最喜欢看到的。
  • 在视觉展现上能够用尽量少的样式去实现设计目的。
  • Token'原本的意思是'令牌，指令'，与 Design 连在一起指'设计变量'。
  • 主题皮肤的替换可以用在两个维度，一是浅色模式和暗色模式的替换，二是不同品牌色之间的替换。
