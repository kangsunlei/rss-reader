📝 详细摘要

文章回顾了 RAG 技术自出现以来的发展历程和范式迭代，特别是在大语言模型广泛应用后，RAG 技术在 2024 年呈现爆发态势。文章详细介绍了 RAG 的五大范式，从 NaiveRAG 到 AdvancedRAG，再到 ModularRAG 和 GraphRAG，以及最新的 AgenticRAG 范式。AgenticRAG 融合了数据库、模型微调、逻辑推理和智能体等多种技术，能够适应各种复杂灵活的任务场景。此外，文章还梳理了 RAG 领域的关键进展，并总结了工程应用中常见的 RAG 系统构建工具，旨在为研究人员和开发者提供 RAG 技术的全面了解，并为工程实践提供参考。

💡 主要观点

  1. RAG 技术通过多轮范式迭代，已从简单形态发展为复杂完善的系统。 RAG 系统经历了从 NaiveRAG 到 AgenticRAG 的演变，融合了多种技术，使其能够应用于各种复杂场景。
  2. AgenticRAG 是 RAG 技术集大成者，具备自适应性，是未来发展方向。 AgenticRAG 结合了 ReACT 的推理能力和 Agent 的任务执行能力，创建了一个动态自适应系统，能主动规划、执行并调整方法以独立解决问题。
  3. 模块化 RAG 架构具有更强的适应性和多功能性。 模块化 RAG 架构超越了前两种 RAG 范式，具有更强的适应性和多功能性。它采用了多种策略来改进其组件，支持顺序处理，也支持跨组件的集成端到端训练。
  4. GraphRAG 利用实体间的结构信息，实现更精确、更全面的检索，捕捉关系知识。 GraphRAG 通过结合图数据库中的结构化信息来增强 LLMs 的输出，实现更精确、更全面的检索。
