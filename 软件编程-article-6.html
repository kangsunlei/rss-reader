ORiGAMi：一种面向文档模型的机器学习架构

3/11/2025, 12:34:57 PM

📌 一句话摘要

ORiGAMi 是一种基于 Transformer 的机器学习架构，它通过直接学习半结构化数据，从而避免了手动转换 JSON 文档的需求。

📝 详细摘要

ORiGAMi 由 MongoDB 的机器学习研究小组开发，是一种新颖的基于 Transformer 的架构，用于对半结构化数据进行监督学习。它将文档转换为键值对序列和结构化标记，使用改进的 Transformer 模型直接从 JSON 文档进行预测，避免了传统机器学习方法的扁平化处理和特征提取。该架构包括确保生成有效文档的措施和一种尊重 JSON 中键值对顺序不变性的位置编码策略。ORiGAMi 在 [github.com/mongodb-labs/origami](http://github.com/mongodb-labs/origami) 上开源，可用于用户细分。

💡 主要观点

  1. ORiGAMi 采用创新的分词方法，将文档转化为键值对序列 通过键值对和结构化标记对 JSON 文档进行序列化，作为 Transformer 模型的输入，使其能够理解文档的结构和内容。
  2. ORiGAMi 通过改进的 Transformer 架构，确保生成符合 JSON 规范的有效文档 该架构包含确保模型生成有效、格式良好的文档的保护措施，并采用尊重 JSON 中键值对顺序不变性的位置编码策略。
  3. ORiGAMi 可以直接在原始文档上训练，无需特征工程 使用户可以直接使用现有的带标签的原始文档进行训练，保留嵌套结构和数组的完整上下文，无需手动进行特征提取。

💬 文章金句

  • 文档模型已被证明是现代应用程序模式下的最佳选择。
  • ORiGAMi 能够直接从半结构化文档进行预测，无需表格数据表示所需的复杂扁平化处理和手动特征提取。
  • 通过将分类重新定义为下一个分词预测任务，ORiGAMi 可以预测文档中的任何字段，包括数组和嵌套子文档等复杂类型。
