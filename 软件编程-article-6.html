隆重推出：Visual Studio 中 GitHub Copilot 代码补全的代码引用功能

2/19/2025, 4:40:08 PM

📌 一句话摘要

Visual Studio 的 GitHub Copilot 通过引入代码引用功能，增强了代码补全的透明度，辅助开发者识别并规避潜在的许可证风险。

📝 详细摘要

文章介绍了 Visual Studio 2022 17.13 中 GitHub Copilot 代码补全新增的代码引用功能。旨在提高代码补全透明度，消除开发者对代码来源和版权风险的顾虑。代码补全与公共 GitHub 仓库匹配时，将通过 Toast 通知提醒。用户可据此决定是否保留或修改代码，以及是否需要添加代码归属声明。文章还介绍了如何配置代码引用功能，允许用户选择阻止或允许匹配公共代码的建议，从而降低因使用公共代码而产生的潜在法律风险。

💡 主要观点

  1. 用户可以配置 Copilot，允许或阻止匹配公共代码的建议 用户可在 GitHub 设置中调整 Copilot 策略，选择是否接受公共代码建议。
  2. 代码引用功能增强了 Copilot 代码补全的透明度 开发者可以查看代码补全是否与公共代码匹配，并获取许可证信息和 GitHub 链接，从而做出更明智的决策。
  3. 代码引用功能主要适用于已接受的 Copilot 代码补全 用户自己编写的代码、修改或放弃的代码补全建议不会进行公共代码匹配检查，减少了不必要的干扰。

💬 文章金句

  • 首次在解决方案中接受与公共 GitHub 仓库匹配的代码补全时，编辑器右下角将显示 Toast 通知。
  • 通过 Toast 通知，可在 GitHub Copilot 输出窗口查看代码匹配项，包括许可证信息和 GitHub.com 链接（指向原始公共代码文件）。
  • 在此更改之前，具有公共代码匹配的 Copilot 代码补全会被自动阻止。现在，开发人员可以选择访问更多代码补全，并接收有关任何公共代码匹配的足够信息，助力开发者做出更明智的决策。
