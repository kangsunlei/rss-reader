Node.js — Node v23.8.0

2/13/2025, 5:44:50 PM

📌 一句话摘要

Node.js v23.8.0 发布，引入系统 CA 证书支持、URL Pattern API、zstd 压缩算法，改进线程命名并更新时区数据。

📝 详细摘要

Node.js v23.8.0 版本带来了多项重要更新，旨在提升安全性、性能，并优化开发者工作流程。该版本支持在 macOS 和 Windows 上使用系统 CA 证书存储，通过新增 `--use-system-ca` 命令行标志实现。同时，引入了 URL Pattern API 的实现，该 API 可用于 URL 匹配。此外，Node.js 现在支持 Zstandard (zstd) 压缩算法，为 `node:zlib` 模块添加了 zstd 流压缩和解压缩相关的 API。为了改善调试体验，Node.js 进程创建的线程现在可以命名。最后，时区数据已更新至 2025a。

💡 主要观点

  1. 支持系统 CA 证书，增强安全性 通过 `--use-system-ca` 标志，Node.js 可以使用系统信任的 CA 证书，提升应用安全性，简化证书管理。
  2. 引入 URL Pattern API，提升路由匹配能力 URL Pattern API 提供了更强大、灵活的 URL 匹配功能，方便开发者处理复杂的路由规则、参数提取和 URL 重写等需求。
  3. 支持 zstd 压缩算法，优化性能 新增的 zstd 压缩算法支持，为 Node.js 应用提供了更高的压缩率和更快的压缩速度，优化数据传输和存储性能。
  4. 改进线程命名，提升调试体验 Node.js 线程现在可以命名，使得开发者在调试多线程应用时，能够更清晰地识别和管理各个线程。这有助于提高调试效率。

💬 文章金句

  • 此版本添加了 `--use-system-ca` 命令行标志，该标志指示 Node.js 使用系统存储中存在的受信任 CA 证书，以及 `--use-bundled-ca`、`--use-openssl-ca` 选项。
  • URL Pattern API 的实现现已可用。
  • Node.js 现在支持 Zstandard (zstd) 压缩算法。
  • Node.js 进程创建的线程现在已命名，以改善调试体验。
