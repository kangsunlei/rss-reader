ReSharper 和 Rider 2024.3：新 Bug 修复更新

2/28/2025, 10:05:24 AM

📌 一句话摘要

ReSharper 和 Rider 2024.3.6 发布，修复了包括 xUnit 测试问题在内的多个错误。

📝 详细摘要

ReSharper 和 Rider 2024.3.6 更新侧重于问题修复，旨在提高开发人员的效率和稳定性。主要修复包括：解决 .NET Framework 下的 xUnit v3 测试失败问题；调试会话启动问题；与 Metal 渲染相关的 macOS 冻结问题；Rider 的“跳转到符号”搜索中的重复项；以及 Qodana 在 Unreal Engine 项目分析期间的挂起问题。有关更改的完整列表，请参阅问题跟踪器。

💡 主要观点

  1. 解决了 .NET Framework 下的 xUnit v3 测试问题 解决了 ReSharper 和 Rider 中的一个问题，即在 .NET Framework 下执行时，xUnit v3 测试显示为“未决”，从而确保了正确的测试执行和结果。
  2. 修复了调试启动和 macOS 冻结问题 解决了调试会话需要多次尝试才能启动的问题。同时，解决了与 Metal 渲染相关的 macOS 冻结问题，从而改善了开发工作流程和稳定性。
  3. 解决了 Rider 中的搜索和代码分析问题 修复了 Rider 的“跳转到符号”搜索结果中出现的重复项。同时，解决了 Qodana 在分析 Unreal Engine 项目时无限期挂起的问题，从而提高了代码导航和分析效率。

💬 文章金句

  • 我们已经解决了一个调试会话需要多次尝试才能启动的问题。
  • 我们已经在更新的 JetBrains Runtime 中修复了与 Metal 渲染相关的 macOS 冻结问题，解决了 `MTLRenderQueue.flushNow` 的问题。
  • 我们修复了一个错误，该错误导致重复项出现在 Rider 的“跳转到符号”搜索结果中。
