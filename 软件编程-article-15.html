📝 详细摘要

文章深入解析了 CSS 新特性 Anchor Positioning 锚点定位，旨在解决传统 Popover 组件中依赖 JavaScript 计算位置的痛点，从而提升性能。首先，文章通过一个简化的 Popover 示例，展示了如何使用 `anchor-name`、`position-anchor` 和 `anchor()` 等核心 API 改变元素的定位基准，实现元素间的动态定位。其次，文章介绍了 `@position-try` 规则和 `position-try-fallbacks` 属性，阐述了如何利用它们实现智能边界处理，使 Popover 在滚动过程中自动调整位置，保持在可视区域。最后，文章总结了 Anchor Positioning 在简化 Popover 实现和提升用户体验方面的优势。同时，文章也提及了 Anchor Positioning 的局限性或未来发展方向，例如浏览器兼容性问题。

💡 主要观点

  1. 掌握 `anchor-name`、`position-anchor`、 `anchor` 等核心 API 至关重要 理解这些 API 的作用和用法，能够灵活运用 Anchor Positioning 解决各种定位问题，提高开发效率。
  2. Anchor Positioning 锚点定位简化了 Popover 组件的实现，提升性能 通过 CSS 即可实现动态改变元素定位基准，无需依赖 JavaScript 计算位置，降低了开发复杂性，提升了性能。
  3. `@position-try` 和 `position-try-fallbacks` 实现了智能边界处理 使得 Popover 组件在超出视窗时，能够自动调整位置，保持在可视区域，提升了用户体验。

💬 文章金句

  • 而锚点定位的属性的核心作用就是，能够改变元素定位的基准，增强元素的绝对定位的能力。
  • Anchor Positioning（锚点定位）允许我们基于其它锚点元素的位置和尺寸去定位上下文，而不是传统意义上的基于父元素去进行绝对定位。
  • 令人振奋的是，现在，CSS 的 Anchor Positioning 锚点定位同样支持这种 **智能边界处理**，在锚点定位中，我们称之为候补位置。
  • 综上所述，到今天，我们已经可以利用 CSS 锚点定位大致实现一个极简版的 Popover 弹窗，并且可以满足大部分场景。
