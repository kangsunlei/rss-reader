Cursor Composer System Prompt

2/14/2025, 3:51:24 AM

📌 一句话摘要

本文介绍了 Cursor 编辑器中用于优化 AI 代码编辑体验的 Composer 功能 System Prompt，该 Prompt 通过特定格式的代码块实现精准代码更新。

📝 详细摘要

本文详细解读了 Cursor 编辑器中 Composer 功能使用的 System Prompt。该 Prompt 主要用于指导 AI 助手（基于 Claude 3.5 Sonnet）在代码编辑任务中的行为，旨在解决 AI 代码编辑中意图表达不清晰和编辑器应用修改不准确的问题。核心要点包括：使用 Markdown 格式回复、通过特定格式的代码块描述代码变更（包括添加 `// ... existing code ...` 标记来保留未修改的代码）、创建新文件时提供完整文件内容、以及根据用户语言进行回复。该 Prompt 旨在提升 AI 代码编辑的准确性和用户体验，确保 AI 助手能够清晰地表达代码修改意图，并帮助编辑器正确地应用这些修改。

💡 主要观点

  1. 使用特定格式的代码块精确描述代码变更，消除 AI 代码编辑中的歧义。 通过 `language:path/to/file` 格式的代码块，并使用 `// ... existing code ...` 标记未修改的代码，确保 AI 助手清晰地表达代码修改意图，并帮助编辑器正确应用这些修改，从而避免潜在的错误和不一致。
  2. System Prompt 强调在代码编辑时保留上下文的重要性。 AI 助手在提供代码修改建议时，不仅要包含修改后的代码，还要展示修改代码所在的函数以及周围的代码，以便用户更好地理解修改的上下文。
  3. System Prompt 指导 AI 助手创建新文件时提供完整内容。 当 AI 助手建议创建新文件时，必须提供新文件的完整内容，确保编辑器能够完整地创建文件，避免遗漏关键信息。

💬 文章金句

  • The assistant will specify all unchanged regions (code and comments) of the file with "// ... existing code ..." comment markers.
  • The assistant will rewrite the entire file only if specifically requested.
  • The assistant will always provide a brief explanation of the updates， unless the user specifically requests only the code.
