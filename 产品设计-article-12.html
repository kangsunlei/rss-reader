DeepSeek 的“服务器繁忙”让所有人抓狂，背后究竟是怎么回事

2/14/2025, 6:02:15 PM

📌 一句话摘要

DeepSeek 因用户激增导致服务卡顿，算力不足和优化滞后是主因，厂商支持未能有效缓解。

📝 详细摘要

文章剖析了 DeepSeek 模型因用户量激增导致服务器频繁繁忙甚至宕机的现象。尽管 DeepSeek 推出了对标 GPT-4o 和 OpenAI o1 的语言模型 V3 和 R1，并吸引了众多云服务商和芯片厂商的支持，但卡顿问题并未得到有效解决。文章分析指出，DeepSeek 的卡顿主要源于推理算力储备不足、模型结构特殊（大模型 + MOE 架构）导致优化困难、以及上线前优化准备不足等因素。此外，黑客攻击也加剧了卡顿现象。文章还对比了 OpenAI 与 DeepSeek 在资源和应对策略上的差异，并提出了 DeepSeek 可能采取的解决方案，如实施付费机制、与云服务商合作等。DeepSeek 在训练成本上具有优势，但推理优化面临挑战。文章最后指出，DeepSeek 短期内可能不会急于解决卡顿问题，用户可能需要长期面对“服务器繁忙”的界面。

💡 主要观点

  1. DeepSeek 卡顿主因是推理算力储备不足，难以应对用户量爆炸式增长。 DeepSeek 在训练成本侧节约的算力，与推理成本侧骤增的算力不匹配，导致应答环节算力不足，用户体验下降。
  2. DeepSeek 模型结构特殊，大模型 + MOE 架构，导致服务商优化需要时间。 服务商虽有算力，但模型优化需要时间，且需兼顾其他模型，分配给 R1 的卡量有限，无法提供稳定服务。
  3. DeepSeek 卡顿本质在于私有云没有做好。 私有云在应对突发流量和保障服务稳定性方面存在挑战，导致 DeepSeek 无法提供稳定可靠的服务。
  4. DeepSeek 短期内可能不会急于解决卡顿问题，用户或需长期面对繁忙界面。 作为追逐 AGI 的公司，DeepSeek 可能更关注长期发展，而非短期用户体验，商业模式梳理清晰后才会考虑租赁服务。

💬 文章金句

  • DeepSeek 频频回复的'服务器繁忙，请稍后再试'，正在让各地用户抓狂。
  • DeepSeek 从输出 token 的推理成本上看是比 OpenAI 同类模型 o1 便宜 90% 以上，这个跟大家的计算偏差不大。
  • R1 热度保持在高位，服务商需要兼顾接入的其他模型，能提供给 R1 的卡很有限，R1 的热度又高，谁家一上 R1，又以相对较低的价格提供，就会被冲垮。
